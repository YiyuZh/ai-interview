## 项目简介

基于大模型的 AI 面试模拟平台。用户上传 PDF 简历，AI 自动解析简历内容并生成针对性面试题目，通过多轮对话模拟真实面试场景，实时流式输出评语，最终生成综合评估报告。

### 功能特性

- 简历上传与 AI 智能解析
- AI 模拟面试
- 实时评分与综合评估报告
- SSE 流式输出
- JWT 登录认证
- 后台管理系统

### 技术栈

| 模块 | 技术 |
|------|------|
| 后端框架 | FastAPI（异步） |
| 前端框架 | Vue 3 + Vite |
| AI 模型 | DeepSeek API（兼容 OpenAI SDK） |
| 数据库 | PostgreSQL 16 |
| 缓存/消息队列 | Redis 7 |
| 异步任务 | Celery（Worker + Beat） |
| ORM | SQLAlchemy 2.0 + asyncpg |
| 容器化 | Docker + Docker Compose |
| 网关 | Caddy |

## 第一步：云服务器环境准备

`[替换原文档“第一步：VM 环境准备”]`

### 1.1 登录服务器

```bash
ssh root@<SERVER_PUBLIC_IP>
```

如果你使用普通账号：

```bash
ssh <YOUR_USER>@<SERVER_PUBLIC_IP>
```

### 1.2 更新系统并安装基础工具

```bash
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg lsb-release git ufw
```

### 1.3 安装 Docker 与 Docker Compose V2

```bash
sudo apt-get install -y docker.io docker-compose-plugin
sudo systemctl enable docker
sudo systemctl start docker
sudo usermod -aG docker $USER
newgrp docker
```

验证：

```bash
docker --version
docker compose version
```

### 1.4 配置 Docker 镜像加速

如果服务器拉镜像较慢，执行：

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<'EOF'
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me",
    "https://docker.m.daocloud.io"
  ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 1.5 配置安全组和防火墙

腾讯云安全组只放行以下端口：

- `22/tcp`
- `80/tcp`
- `443/tcp`

不要放行以下端口：

- `5432`
- `5434`
- `6379`
- `6382`
- `8001`
- `8006`

如果服务器启用了 `ufw`，执行：

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw --force enable
sudo ufw status
```

### 1.6 准备部署目录

```bash
sudo mkdir -p /srv/apps
sudo chown -R $USER:$USER /srv/apps
cd /srv/apps
```

### 1.7 拉取 AI-Interview 项目代码

```bash
cd /srv/apps
git clone <AI_INTERVIEW_GIT_URL> ai-interview
```

如果你不是通过 Git 部署，而是手动上传代码，请把整个 `ai-interview` 目录上传到：

```bash
/srv/apps/ai-interview
```

## 第二步：接入现有 HireMate 网关

`[新增]`

当前服务器上已经有 `HireMate` 运行，并且 `HireMate` 现有的 `Caddy` 已占用 `80/443`。  
因此本次不再新增第二个公网网关，而是让 `AI-Interview` 通过共享 Docker 网络挂到现有 `HireMate Caddy` 后面。

### 2.1 备份 HireMate 配置

```bash
cd /srv/apps/HireMate
cp docker-compose.yml docker-compose.yml.bak-$(date +%F-%H%M%S)
cp Caddyfile Caddyfile.bak-$(date +%F-%H%M%S)
```

### 2.2 创建共享网关网络

```bash
docker network create shared_gateway || true
docker network ls | grep shared_gateway
```

### 2.3 修改 HireMate 的 docker-compose.yml

打开文件：

```bash
cd /srv/apps/HireMate
nano docker-compose.yml
```

把 `caddy` 服务改成下面这样：

```yaml
services:
  caddy:
    image: caddy:2
    container_name: hiremate-caddy
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
      - "443:443/udp"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config
    depends_on:
      - hiremate
    networks:
      - default
      - shared_gateway

volumes:
  hiremate_data:
  hiremate_mysql_data:
  caddy_data:
  caddy_config:

networks:
  shared_gateway:
    external: true
    name: shared_gateway
```

保存后仅重建 `caddy` 服务：

```bash
cd /srv/apps/HireMate
docker compose up -d caddy
docker compose ps
```

这一步不会改动 `HireMate` 的业务容器逻辑，只是让 `caddy` 额外加入共享网络。

## 第六步：配置 Caddy 统一入口

`[新增]`

### 6.1 打开现有 HireMate 的 Caddyfile

```bash
cd /srv/apps/HireMate
nano Caddyfile
```

把配置调整为下面这种形式。  
如果你当前 `HireMate` 的域名和这里不同，请替换占位符：

```caddy
{
    email <CADDY_ACME_EMAIL>
}

<PRIMARY_DOMAIN> {
    encode gzip zstd
    reverse_proxy ai-interview-portal:80
}

<HIREMATE_DOMAIN> {
    encode gzip zstd
    reverse_proxy hiremate:8501
}

<AI_INTERVIEW_DOMAIN> {
    encode gzip zstd

    @backend path /api/* /uploads/* /client/* /backoffice/*
    handle @backend {
        reverse_proxy ai-interview-app:8001
    }

    handle {
        reverse_proxy ai-interview-frontend:80
    }
}

<AI_INTERVIEW_ADMIN_DOMAIN> {
    encode gzip zstd

    @backend path /api/* /uploads/* /client/* /backoffice/*
    handle @backend {
        reverse_proxy ai-interview-app:8001
    }

    handle {
        reverse_proxy ai-interview-admin:80
    }
}
```

### 6.2 重载 Caddy

```bash
cd /srv/apps/HireMate
docker compose up -d caddy
docker compose logs caddy --tail 100
```

如果日志中没有明显报错，并且域名已经正确解析到服务器公网 IP，那么 Caddy 会自动申请并续期 HTTPS 证书。

## 端口说明

| 服务 | 容器端口 | 宿主机端口 | 是否公网暴露 |

确认原系统仍可正常打开和使用。

### 7. 验证数据库和 Redis 未暴露公网

```bash
ss -lntp | grep -E '5432|5434|6379|6382|8001|8006'
```

期望结果：

- 不应出现对公网开放的 `5432/6379/8001/8006`
- 对外只应保留 `80/443`

## 故障排查

### AI-Interview 后端启动失败

```bash
cd /srv/apps/ai-interview/ai-interview-backend
docker compose -f docker-compose.yml -f docker-compose.prod.yml logs app --tail 100
```

常见原因：

- `.env` 中数据库账号或密码错误
- PostgreSQL 还没 ready
- Redis 密码配置不一致
- `DEEPSEEK_API_KEY` 未填写

### 数据表不存在

```bash
cd /srv/apps/ai-interview/ai-interview-backend
docker compose -f docker-compose.yml -f docker-compose.prod.yml exec app alembic upgrade head
```

如果仍有问题，再检查：

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml exec postgres \
  psql -U <AI_INTERVIEW_DB_USER> -d <AI_INTERVIEW_DB_NAME> -c "\dt"
```

### 前端页面打开但接口报错

检查：

- `Caddyfile` 中 `AI_INTERVIEW_DOMAIN` 和 `AI_INTERVIEW_ADMIN_DOMAIN` 是否配置正确
- `.env` 中 `BACKEND_CORS_ORIGINS` 是否写对
- `docker compose ps` 是否显示 `app/frontend/admin/portal` 都在运行

### HTTPS 证书申请失败

检查：

- 域名是否已解析到服务器公网 IP
- 80 和 443 是否在腾讯云安全组中已放行
- 服务器是否能访问外网
- Caddy 日志中是否有 ACME 申请报错

查看日志：

```bash
cd /srv/apps/HireMate
docker compose logs caddy --tail 100
```

### 修改 .env 后配置不生效

`.env` 修改后，必须重新创建容器：

```bash
cd /srv/apps/ai-interview/ai-interview-backend
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
```

仅执行 `restart` 通常不会重新读取构建参数。

## 回滚步骤

### 回滚原则

- 优先先恢复 `HireMate` 网关配置
- 不要直接删除数据库 volume
- 如需回滚数据库结构，只使用 `Alembic downgrade`

### 1. 回滚 Caddy 配置

```bash
cd /srv/apps/HireMate
cp Caddyfile.bak-<你的备份时间戳> Caddyfile
docker compose up -d caddy
```

### 2. 停止 AI-Interview

```bash
cd /srv/apps/ai-interview/ai-interview-backend
docker compose -f docker-compose.yml -f docker-compose.prod.yml down
```

### 3. 如果需要回滚代码版本

```bash
cd /srv/apps/ai-interview
git log --oneline
git checkout <OLD_COMMIT_ID>

cd /srv/apps/ai-interview/ai-interview-backend
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
```

### 4. 如果需要回滚数据库迁移

先查看版本：

```bash
cd /srv/apps/ai-interview/ai-interview-backend
docker compose -f docker-compose.yml -f docker-compose.prod.yml exec app alembic history
```

执行回滚：

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml exec app \
  alembic downgrade <REVISION_ID>
```
