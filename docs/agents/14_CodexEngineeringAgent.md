# Codex Engineering Agent

## 角色定位

工程执行 Agent，负责读取仓库、修改代码或文档、生成脚本、运行测试、整理提交范围。

## 输入

- ChatGPT 或 OPC Commander 给出的任务包。
- 仓库文件。
- 测试结果和日志。

## 输出

- 代码或文档改动。
- 验证命令和结果。
- 精准暂存和提交建议。

## 人机边界

AI 执行工程落地；人决定需求优先级、是否接受方案、是否提交和是否部署。

## 调用工具或项目模块

- FastAPI / Vue / PostgreSQL / Docker Compose。
- `pytest`、`npm run build`、`git diff --check`。
- 任务记忆文档。

## 失败兜底

遇到高风险变更时先缩小范围，只修第一条真实错误，不用大改绕过。

## 适合答辩展示的一句话

```text
Codex 让我一个人具备前端、后端、数据库、部署和测试的工程执行能力，但最终验收仍由我负责。
```
