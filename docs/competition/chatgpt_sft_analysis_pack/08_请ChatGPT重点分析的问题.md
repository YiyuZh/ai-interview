# 请 ChatGPT 重点分析的问题

## 1. 当前能不能开始训练

请判断当前项目是否可以直接创建 OpenAI SFT job。

需要重点考虑：

- 本地代码还领先远端，服务器是否能拉到最新脚本。
- 是否已配置官方 OpenAI Key。
- 是否已有可 fine-tune 的 base model。
- 是否已有至少 3 条真实授权且人工复核的样本。
- 是否已有至少 10 条总训练样本。
- 是否已有至少 5 条验证集样本。

## 2. 第一版 SFT 任务是否合理

当前建议第一版只训练“面试追问生成”任务。

请分析：

- 这个任务是否足够窄。
- 是否比训练完整报告生成更适合第一版。
- 输入字段应该包含哪些内容。
- 输出格式应该如何设计，才能方便 eval。
- 是否需要把反例样本单独用于评估，而不是混入正向 SFT。

## 3. 样本量和样本结构怎么安排

请给出建议：

- 最低可跑样本规模。
- 比赛答辩更有说服力的样本规模。
- 真实授权样本和构造样本的比例。
- 三岗位样本如何分布。
- 训练集和验证集如何拆分。
- 每条样本的 metadata 应该记录什么。

## 4. Eval 怎么设计

请设计一套轻量但可信的 eval：

- 每条 holdout 样本如何评分。
- 是否需要人工评分表。
- 是否需要模型自评。
- 如何避免模型自评自嗨。
- 如何呈现 base model 和 fine-tuned model 的差异。
- 哪些结论能写进答辩，哪些只能作为内部观察。

## 5. 答辩怎么讲更有竞争力

请给出答辩建议：

- 如何把“微调准备链路”讲得像一个真实落地的工程闭环。
- 如何解释为什么先做数据闭环，再做模型训练。
- 如何避免被评委追问“你是不是只是调 API”。
- 如何展示岗位画像、RAG、授权样本、人工评分和 SFT 的关系。
- 如果真实训练还没跑完，如何诚实但有竞争力地表达。

## 6. 下一步执行计划

请给出 7 天执行计划：

- 每天目标。
- 每天产出。
- 哪些命令要跑。
- 哪些文件要保存。
- 哪些结果能用于答辩。
- 哪些结果不能夸大。
