# 项目概况

## 1. 项目定位

项目名称：职启智评  
当前定位：就业能力诊断与提升平台  
比赛场景：机器人与人工智能大赛人工智能创新赛答辩与演示

项目不再定位为单一的“AI 面试官”，而是围绕求职准备建立一条完整闭环：

```text
简历解析
-> 目标岗位
-> 岗位画像 / 岗位知识库
-> 简历证据评分
-> 能力差距诊断
-> 学习任务
-> AI 模拟面试
-> 报告生成
-> 训练复盘
-> 本次案例授权
-> 后台人工评分沉淀
-> 评测与微调样本准备
```

## 2. 核心价值

项目要解决的问题不是“让大模型随便问几个面试题”，而是：

- 大学生不了解目标岗位真实能力要求。
- 简历中很多能力描述缺少证据。
- 通用 AI 面试容易泛泛而谈，缺少岗位画像约束。
- 面试报告和学习任务如果没有统一评价口径，容易前后不一致。
- 真实训练数据如果没有授权、脱敏和人工复核，不能安全进入后续模型优化。

## 3. 已有产品能力

当前项目已经具备以下基础能力：

- 用户端上传简历并解析 PDF 文本。
- 生成结构化简历和解析质量信息。
- 基于目标岗位进行能力诊断。
- 基于证据状态生成简历润色建议。
- 生成学习任务和训练路线。
- 根据岗位画像和能力缺口启动模拟面试。
- 生成面试报告和训练复盘。
- 用户可对本次案例做去标识化数据贡献授权。
- 后台可进行案例筛选、人工评分和样本沉淀。
- 评测样本与微调准备链路已经搭建。

## 4. 技术架构

当前主要技术栈：

- 前端用户端：Vue
- 后台管理端：Vue
- 后端：FastAPI
- 数据库：PostgreSQL
- 缓存与异步任务：Redis / Celery
- 部署：Docker Compose
- 大模型接入：OpenAI-compatible API，当前微调路线以 OpenAI SFT 为目标

## 5. 微调相关的产品基础

项目已经为微调做了几个关键准备：

- 用 `resume_evaluation_snapshot` 统一简历评分、润色、学习计划、面试、报告和复盘口径。
- 通过岗位知识库和岗位画像约束大模型，不把岗位资料写成候选人的真实经历。
- 通过用户授权和去标识化规则控制真实数据是否能进入评测和微调准备。
- 通过后台人工评分提高训练样本质量。
- 通过导出和转换脚本，把业务样本转成 OpenAI chat fine-tuning JSONL。
