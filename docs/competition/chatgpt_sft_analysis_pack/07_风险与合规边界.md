# 风险与合规边界

## 1. 隐私授权边界

真实用户数据进入评测、人工评分沉淀或微调准备前，必须获得明确授权。

当前采用两层口径：

- 基础隐私协议：用于提供简历解析、能力诊断、简历润色、模拟面试、报告和训练复盘等核心服务。
- 可选数据贡献授权：用于系统评测、比赛材料、质量改进、人工评分和数据集沉淀。

可选数据贡献授权默认不勾选，不影响用户使用核心功能。

## 2. 去标识化边界

当前应使用“去标识化”或“脱敏”表述，不应写成“完全匿名”。

必须去除或遮挡直接身份标识：

- 姓名。
- 手机号。
- 邮箱。
- 证件号。
- 学号。
- 详细住址。
- 文件名中的个人标识。

在明确告知和授权前提下，为保证岗位诊断质量，可以保留学校、专业、项目经历、技能、目标岗位和面试回答等信息。

## 3. 构造样本边界

构造样本可以用于：

- 补齐输出格式。
- 覆盖三类岗位。
- 训练模型遵守证据约束。
- 训练模型稳定生成追问。

构造样本不能用于：

- 证明真实用户效果。
- 冒充真实授权样本。
- 替代 C1/C2/C3 真实闭环。
- 证明系统已经形成大规模真实数据。

## 4. 大模型调用边界

如果使用 OpenAI 官方 API：

- 不应提交 API Key。
- 不应把 Key 发到聊天窗口。
- 不应把包含 Key 的 `.env` 截图。
- 如涉及真实个人信息出境或跨境传输，应另行评估适用规则。

如果使用中转 API：

- 不建议作为正式微调依据。
- 不建议在答辩中说成 OpenAI 官方 fine-tuning。
- 需要额外评估数据处理和隐私风险。

## 5. 答辩诚信边界

答辩中应如实区分：

- 已实现功能。
- 已准备脚本。
- 已启动训练。
- 已完成训练。
- 已完成评估。

没有 `fine_tuned_model` 前，不应把数据准备说成训练完成。

没有 eval 结果前，不应写具体效果结论。

没有真实授权样本统计前，不应宣称真实数据规模。
