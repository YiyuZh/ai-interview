# 一人 OPC 日常作战手册

## 1. 每天固定工作流

每天按 5 步走：

```text
1. 读记忆：PROJECT_MEMORY.md / 任务记录文档.md
2. 定目标：今天只完成一个可验收成果
3. 派 Agent：把任务拆给 Codex / ChatGPT / 业务大模型
4. 收结果：只看交付物、测试结果、截图、日志
5. 写记录：更新任务记忆和下一步
```

## 2. 每天开局提示词

```text
你是我的 OPC 总控助理。请先阅读 PROJECT_MEMORY.md、任务记录文档.md 和 docs/competition/职启智评项目升级流程手册.md，判断当前最高优先级。今天只允许推进一个明确可验收目标。请输出：今日目标、涉及文件、执行步骤、验收标准、禁止动作、完成后需要追加的任务记录模板。
```

## 3. Codex 工作方式

给 Codex 的任务必须包含：

```text
背景
目标
禁止动作
必须阅读文件
需要修改文件
验收命令
任务记录更新要求
```

不要给 Codex 模糊任务：

```text
优化一下项目。
```

要给明确任务：

```text
在 docs/competition/opc 下生成复赛限时实测训练包，不改业务代码；完成后更新 PROJECT_MEMORY.md 和任务记录文档.md。
```

## 4. ChatGPT 工作方式

ChatGPT 适合做：

```text
赛事研究
答辩叙事
评委问答
技术路线比较
PPT 页结构
讲稿压缩
风险边界检查
```

典型提示词：

```text
你是 OPC 赛事答辩教练。请基于赛事公开要求和我的项目状态，把“职启智评”改写成超级个体 AI 工作流项目，重点突出我如何设计人机协同结构，而不是只讲产品功能。
```

## 5. 业务大模型工作方式

业务大模型用于系统内部：

```text
简历解析
简历诊断
岗位画像总结
面试追问
报告复盘
学习计划
```

注意：业务大模型输出必须受证据状态约束。不能把岗位知识库写成候选人真实经历。

## 6. 每周节奏

```text
周一：确定本周目标和比赛材料缺口
周二：Codex 改造系统或生成演示资产
周三：补赛事材料和讲稿
周四：跑 demo、截图、录屏
周五：模拟评委问答
周六：修漏洞和补文档
周日：整理版本、提交、备份
```

## 7. 一人项目最怕什么

```text
1. 同时做太多功能，最后没有一个可展示闭环。
2. 只写 PPT，不保留可运行 trace 和截图。
3. 只让 AI 生成内容，不做人类验收。
4. 不更新任务记忆，导致 Codex 重复走错路。
5. 答辩只讲产品，不讲 OPC 主体能力。
```

## 8. 一人 OPC 的核心证明材料

```text
1. 任务记录文档：证明你长期用 AI 推进项目。
2. Agent 角色卡：证明你设计了人机协同结构。
3. Agent Trace：证明工作流不是空话。
4. 演示页面：证明项目可运行。
5. 复赛实测训练包：证明你能现场构建 AI 工作流。
6. 三个月补实路线：证明项目可持续。
```
