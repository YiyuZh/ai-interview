# 演示资产生成规范

## 1. 演示资产的目标

演示资产不是为了冒充真实用户数据，而是为了让评委快速看懂：

```text
一个人如何用 AI Agent 编排高校就业诊断服务
一次诊断流程如何从输入流到输出
哪些环节由 AI 做，哪些环节由人审核
```

## 2. 必须生成的资产

```text
artifacts/opc/
  opc_workflow_trace.md
  opc_workflow_trace.json
  opc_human_ai_division.md
  opc_ppt_assets.md
  opc_live_test_summary.md
```

复用现有：

```text
demo_cases/
artifacts/agent_trace/
artifacts/eval/
artifacts/sft_preview/
```

## 3. 三岗位演示样本

固定三类：

```text
C1：Python 后端开发工程师
C2：产品助理 / 产品经理实习生
C3：人力资源专员
```

所有演示样本必须标记：

```json
{
  "sample_origin": "demo_constructed",
  "for_training": false,
  "for_competition_demo": true
}
```

## 4. OPC Trace 标准结构

```json
{
  "opc_commander": {
    "human_decision": "选择高校就业服务作为垂直场景",
    "reason": "需求明确、项目已有基础、适合轻量化服务"
  },
  "agents": [
    {
      "name": "Resume Evidence Agent",
      "input": "去标识化简历摘要",
      "output": "证据状态表",
      "human_boundary": "不自动断言真实掌握，只给待验证项"
    }
  ],
  "final_outputs": [
    "能力诊断",
    "简历润色",
    "面试追问",
    "学习任务",
    "报告复盘",
    "Eval Preview",
    "SFT Preview"
  ]
}
```

## 5. PPT 可用图

Codex 应生成 Mermaid 源码：

```text
AI 工作流总图
人机协同分工图
数据飞轮图
三个月补实路线图
复赛限时实测解题流程图
```


## 7. 命名规范

```text
opc_*.md
opc_*.json
opc_*.png
opc_*.mp4
```

不要命名成：

```text
真实微调结果
真实用户数据集
已训练模型效果
```

除非后续真的完成相关结果。
