# 原项目到 OPC 创新点映射表

## 1. 通俗说明：最后跟原项目相比改了什么

原项目已经是一个就业能力诊断平台。OPC 改造不是推翻原项目，而是做了三类增强：

```text
第一，把产品功能重新组织成 AI 工作流。
第二，把你本人和 AI 的协同关系讲清楚。
第三，把项目从“平台展示”变成“超级个体能力展示”。
```

## 2. 原项目功能 vs OPC 表达

| 原项目功能 | OPC 版表达 | 创新点 |
|---|---|---|
| 上传简历 | Resume Evidence Agent 输入 | AI 自动处理非结构化资料 |
| 简历解析 | 简历结构化理解 Agent | 把 PDF 转成可计算证据 |
| 岗位知识库 | Role Profile RAG Agent | 岗位知识资产化、可切片、可召回 |
| 能力诊断 | Gap Diagnosis Agent | 简历证据和岗位要求对齐 |
| 简历润色 | Resume Polish Agent | 不造假，只强化证据表达 |
| AI 面试 | Interviewer Agent | 证据追问，不问泛泛模板题 |
| 学习任务 | Learning Plan Agent | 把诊断转成行动计划 |
| 报告复盘 | Report Review Agent | 把训练结果沉淀成下一轮目标 |
| 后台人工评分 | Human-in-the-loop Governance | 人类校准 AI 输出 |
| SFT Preview | Post-training Dataset Agent | 为后续持续优化准备数据 |
| 任务记录文档 | OPC Governance Log | 证明一人 + AI 的持续工作流 |

## 3. 原来像产品，现在像 OPC

原项目证明：

```text
我能开发一个就业能力诊断系统。
```

OPC 改造证明：

```text
我能用 AI 工具链设计、开发、部署、展示和治理一个垂直行业服务单元。
```

## 4. 最核心的创新点

### 创新点 1：AI 原生就业服务工作流

不是单点 AI 问答，而是：

```text
简历 -> 岗位 -> 证据 -> Gap -> 润色 -> 学习 -> 面试 -> 报告 -> 评分 -> 样本
```

### 创新点 2：OPC 人机协同结构

你本人是总控，AI Agent 是执行单元。

### 创新点 3：简历证据闭环

系统区分：

```text
direct / indirect / claimed_only / missing
```

减少“简历写了就算会”的误判。

### 创新点 4：岗位画像 RAG 与多信号路由

岗位知识库不是摆设，而是用于面试追问、润色建议和报告复盘。

### 创新点 5：轻量化创新单元

不训练大型底层模型，不建大团队，用开源框架、API、大模型、Codex 和自动化脚本完成高效迭代。

### 创新点 6：可补实的数据飞轮

```text
用户授权 -> 去标识化 -> 人工评分 -> Eval -> SFT-ready -> 下一轮优化
```

## 5. 你最后要会说的通俗版

```text
原来我的项目像一个 AI 面试网站。现在我把它升级成了一个 OPC 超级个体工作流样板：我一个人负责决策和审核，AI Agent 分别承担代码、材料、简历、岗位、面试、报告和评估。这样我不是在展示一个功能，而是在展示一个人如何借助 AI 构建一个轻量化就业服务单元。
```
