# 目录结构与脚本规范

## 1. 推荐新增目录

```text
docs/competition/opc/
docs/competition/opc/live_test_drills/
docs/agents/
artifacts/opc/
```

## 2. 推荐新增脚本

```text
ai-interview-backend/app/scripts/generate_opc_assets.py
ai-interview-backend/app/scripts/generate_opc_live_test_pack.py
ai-interview-backend/app/scripts/export_opc_workflow_trace.py
```

## 3. 脚本原则

```text
1. 默认只生成 artifacts，不写业务数据库。
2. 不调用真实付费训练 API。
3. 不读取未授权真实简历。
4. 输出必须带 preview/demo 标识。
5. 可从仓库根目录运行。
```

## 4. 推荐命令

```bash
cd D:/apps/ai-interview
python -m ai-interview-backend.app.scripts.generate_opc_assets
```

如果项目已有根层 `app` shim，则使用：

```bash
python -m app.scripts.generate_opc_assets
```

## 5. 输出文件

```text
artifacts/opc/opc_workflow_trace.md
artifacts/opc/opc_workflow_trace.json
artifacts/opc/opc_ppt_assets.md
artifacts/opc/opc_live_test_summary.md
```

## 6. Git 暂存建议

不要 `git add .`。

推荐精准暂存：

```bash
git add docs/competition/opc
git add docs/agents
git add artifacts/opc/.gitkeep
git add PROJECT_MEMORY.md
git add 任务记录文档.md
git add docs/competition/职启智评项目升级流程手册.md
```

如新增脚本：

```bash
git add ai-interview-backend/app/scripts/generate_opc_assets.py
```

排除：

```text
真实简历原文
.env
API Key
大体积录屏
未脱敏 PDF
临时 .codex_tmp 文件
node_modules / dist
```
