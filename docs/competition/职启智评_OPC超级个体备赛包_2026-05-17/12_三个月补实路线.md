# 三个月补实路线

## 第 1 月：真实闭环补实

目标：把演示链路变成真实样本链路。

任务：

```text
1. 完成 C1/C2/C3 三岗位真实闭环。
2. 每个案例至少 3 轮面试，推荐 5 轮。
3. 完成报告、训练复盘和本次案例数据贡献授权。
4. 后台人工评分：证据对齐、问题质量、报告可执行性、学习任务可执行性。
5. 回填 CSV 和真实闭环记录。
```

交付物：

```text
真实案例闭环验收记录.md
C1_C2_C3_case_records.csv
后台人工评分截图
真实授权样本统计表
```

## 第 2 月：模型与工作流优化

目标：证明工作流不是只会演示，而是可评估可优化。

任务：

```text
1. 补足真实授权样本不少于 10 条。
2. 构造样本补格式和岗位覆盖，但继续标记 constructed。
3. 生成 train/validation JSONL。
4. 如果 OpenAI 条件满足，启动 SFT。
5. 如果 OpenAI 条件不满足，做 Prompt/RAG/Agent 版本 Eval 对照。
```

交付物：

```text
sft_summary.json
manifest.jsonl
pii_scan_report.json
base_vs_agent_eval.md
如果真实 SFT 成功：job_record.json / fine_tuned_model / eval_report.md
```

## 第 3 月：产业场景与 OPC 治理

目标：让项目更贴近赛事后续“场景嵌入实践”。

任务：

```text
1. 找 1 个学院、班级或就业指导场景做白名单试用。
2. 收集 5-10 条反馈。
3. 完善隐私协议、删除机制和数据授权说明。
4. 形成高校就业服务接入方案。
5. 输出升级版 AI 原生结构说明书。
```

交付物：

```text
高校就业服务场景对接方案.md
白名单试用反馈记录.md
升级版AI原生结构说明书.md
OPC主体治理说明.md
三个月实践总结.md
```

## 三个月后可以更强势表达

如果全部完成，可以说：

```text
已完成三岗位真实闭环。
已沉淀真实授权样本和人工评分记录。
已完成 base / agent 或 base / fine-tuned 对照 Eval。
已形成高校就业服务小范围应用验证。
```

```text

```
