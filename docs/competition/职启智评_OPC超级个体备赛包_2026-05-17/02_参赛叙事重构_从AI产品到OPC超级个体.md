# 参赛叙事重构：从 AI 产品到 OPC 超级个体

## 1. 一句话重构

旧说法：

```text
我做了一个 AI 面试官平台。
```

新说法：

```text
我作为一个 OPC 超级个体，设计了一套面向高校就业服务的 AI 原生工作流，用多 Agent 协同完成简历诊断、岗位画像、能力差距分析、简历润色、学习任务、模拟面试、报告复盘和人工评分沉淀。
```

## 2. 参赛项目名称建议

主标题：

```text
职启智评 OPC：一个人 + AI Agents 的高校就业能力诊断与训练服务单元
```

副标题：

```text
基于 Career-AgentOS 的人机协同工作流、岗位画像 RAG 与简历证据闭环
```

更正式版本：

```text
职启智评：面向高校就业服务的 OPC 超级个体 AI 原生工作流系统
```

## 3. 答辩主线

推荐 6 句话主线：

```text
第一，我不是只开发了一个 AI 面试网站，而是把高校就业指导拆成了一套可复用的 AI 工作流。
第二，这套工作流由我作为 OPC 总控，多个 AI Agent 分担资料整理、代码实现、岗位建模、简历诊断、面试追问、报告复盘和答辩材料生成。
第三，AI 负责高频文本处理、代码生成、RAG 检索、初步诊断和文档生成；人负责目标判断、资料真实性、隐私授权、评分标准和最终决策。
第四，系统嵌入大学生求职训练场景，解决岗位认知不清、简历证据不足、面试训练零散和反馈难复盘的问题。
第五，项目采用轻量化技术栈和外部大模型 API，不依赖大团队和重算力，一个人即可构建、部署、演示和持续迭代。
第六，后续三个月将把现有 Preview 链路补实为真实授权样本、人工评分、OpenAI SFT 和可复现 Eval。
```

## 4. 你本人要被塑造成什么角色

你不是“普通开发者”，而是：

```text
OPC Commander / AI 原生就业服务设计者 / 一人产品公司负责人
```

你的职责：

```text
1. 判断高校就业服务场景中什么问题值得解决。
2. 把问题拆成 AI 可以执行的工作流。
3. 设计人和 AI 的边界。
4. 用 Codex 快速实现前后端、数据库、部署和脚本。
5. 用 ChatGPT 完成赛事研究、答辩材料、技术路线和评委问答。
6. 用大模型 API 承担业务中的简历理解、问题生成、报告组织。
7. 通过人工评分和授权样本持续校准系统。
```

## 5. 评委最容易记住的创新点

请反复强调这 4 个词：

```text
工作流
协同边界
垂直嵌入
轻量治理
```

对应解释：

```text
工作流：不是一次 AI 问答，而是从简历到复盘的连续 AI 流程。
协同边界：AI 做高频、重复、生成、检索，人做判断、审核、授权、取舍。
垂直嵌入：不是通用聊天，而是嵌入高校就业服务。
轻量治理：一个人可维护，可部署，可迭代，可审计。
```

## 6. 和原项目的关系

不要说推翻原项目。

要说：

```text
原项目是可运行产品底座，OPC 改造是在它上面增加 AI 原生结构表达、人机协同编排、复赛实测训练和轻量化运营方法。
```

原项目提供：

```text
前端用户端
后台管理端
FastAPI 后端
PostgreSQL 数据库
Redis/Celery 异步能力
岗位画像和知识库
AI 面试和报告
用户授权和后台标注
```

OPC 改造提供：

```text
一人总控结构
AI Agent 分工
工作流 Trace
限时实测训练
答辩主线
三个月补实路线
```
