# 赛事研究与适配判断

## 1. 公开信息要点

根据公开赛事通知，第五届“京彩大创”设置主体赛事和产业应用赛事。其中产业应用赛事聚焦真实产业场景和企业生产实践，引导高校师生技术攻关与成果转化。中关村科学城管理委员会围绕人工智能产业应用需求，举办“京彩AI·智汇全球”北京大学生“超级个体”（OPC）创业大赛。

OPC 专项更关键的不是传统商业计划书，而是：

```text
AI 原生结构设计
人机协同结构设计
垂直行业嵌入能力
轻量化创新单元构建能力
主体治理能力
```

公开通知中对“超级个体”的定义是：在人工智能环境中成长、能够构建人机协同结构并形成轻量化创新单元的青年创新主体；OPC 是 One Person Company，一人公司，为制度性组织形态，强调轻结构运行、独立法人责任与可持续创新能力。

复赛集中考察方向包括：

```text
AI 工作流构建
人机协同逻辑设计
产业应用结构表达
集中限时实测
阶段成果展示
```

## 2. 对职启智评的适配结论

原项目“职启智评”并不是完全不适合 OPC，但如果仍按普通 AI 产品讲，会偏离评审重点。

原项目优势：

```text
1. 已有可运行原型，不只是概念。
2. 有高校就业服务这一垂直场景。
3. 有完整业务闭环：简历解析、岗位画像、证据评分、简历润色、学习任务、AI 面试、报告复盘、人工评分。
4. 有多 Agent / Career-AgentOS 叙事基础。
5. 有 Codex、ChatGPT、DeepSeek/OpenAI API、脚本、部署、测试和材料生成的真实 AI 使用记录。
```

原项目短板：

```text
1. 过于像“我做了一个平台”，不够像“我作为超级个体设计了 AI 工作流”。
2. 产品功能讲得多，人机协同结构讲得少。
3. 技术路线讲得多，OPC 主体治理讲得少。
4. 复赛集中限时实测准备不足。
```

## 3. 最适合的参赛主张

推荐主张：

```text
我是一个 AI 原生超级个体，围绕高校就业服务这一垂直场景，设计并运营了 Career-AgentOS 多 Agent 工作流。它把一个就业咨询小团队的能力拆解为可编排的 AI Agent：简历证据、岗位画像、能力差距、简历润色、面试追问、学习任务、报告复盘、数据治理和评估沉淀。通过人机协同，我一个人可以完成产品设计、代码开发、资料处理、部署验收、答辩材料生成和持续优化。
```

## 4. 推荐赛道定位

优先定位：

```text
“京彩AI·智汇全球”北京大学生“超级个体”（OPC）创业大赛
方向：人工智能与新一代信息技术 / 数字经济与企业服务 / 未来产业与科技服务
应用场景：高校就业服务、大学生求职能力训练、就业指导数字化
```

备选表达：

```text
项目同时具备人工智能赛道属性，但本次参赛重点不是单纯 AI 产品，而是 AI 原生结构、人机协同和轻量化创新单元。
```

## 5. 评委会怎么看你的项目

评委可能不会只问：

```text
你的 AI 面试准不准？
```

更可能问：

```text
你作为一个人如何调用 AI 替代小团队？
你的人机协同边界是什么？
AI 工作流怎么构建？
现场给一个新岗位，你能不能快速搭建新的就业诊断流程？
你怎么保证 AI 不乱编？
你如何沉淀数据和持续优化？
```

所以答辩要从“产品功能说明”转成“OPC 能力证明”。
