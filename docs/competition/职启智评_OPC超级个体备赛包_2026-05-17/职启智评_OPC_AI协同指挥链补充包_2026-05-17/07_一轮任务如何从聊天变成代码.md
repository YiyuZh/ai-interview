# 07 一轮任务如何从聊天变成代码

## 标准流水线

```text
用户目标
-> ChatGPT 赛道/产品/技术分析
-> ChatGPT 生成 Codex 任务包
-> Codex 读取项目记忆
-> Codex 定位代码和文档落点
-> Codex 修改/生成文件
-> Codex 验证
-> Codex 追加任务记录
-> ChatGPT 转成答辩话术
-> 我本人验收
```

## 示例 1：知识库切片能力

你提出：

```text
岗位知识库整份文本注入 prompt 不够专业。请最小改动升级为切片化、结构化元数据、Top-K 召回，失败时回退全文。
```

ChatGPT 拆解：

```text
不引入向量库，先做结构化切片 + 规则召回；新增切片表和服务；保存知识库时自动重建；面试开始和逐轮追问时召回；保留全文回退。
```

Codex 执行：

```text
新增 position_knowledge_base_slice.py
新增 position_knowledge_base_slice_service.py
新增 Alembic migration
修改 interview_service.py
修改 ai_service.py
补测试
更新任务记录
```

答辩转译：

```text
系统不是把整份岗位资料塞给大模型，而是先把岗位资料自动切片并按面试阶段、技能、难度和弱点词路由召回，降低 prompt 噪声，提高追问相关性。
```

## 示例 2：多面试官协同

你提出：

```text
想让项目更像多 Agent 系统，但前端不要改成多个 AI 同时说话，也不要多模型并发。
```

ChatGPT 拆解：

```text
做内部多角色协同：技术深挖、项目追问、表达结构、业务场景、压力质询；对外仍由一个主持人输出。
```

Codex 执行：

```text
ai_service.py 新增 panel 结构化输出
interview_service.py 存 panel_snapshot
评估时输出 panel_views 和 moderator
JSON 失败回退单面试官
```

答辩转译：

```text
系统内部模拟面试委员会多视角评审，对用户侧仍保持一个主持人，降低交互复杂度，同时提升追问质量。
```

## 示例 3：OPC AI 协同材料

你指出：

```text
材料没有讲清 AI 之间如何协调。
```

ChatGPT 拆解：

```text
补 AI 使用顺序、Prompt 链、聊天到任务映射、Codex SOP、现场实测 SOP。
```

Codex 执行：

```text
生成 docs/competition/opc_ai_coordination/*.md
生成 artifacts/opc_ai_coordination/*.sample.*
追加阶段 165 任务记录
```
