# 03 多 AI 角色分工与交接协议

## 1. AI 之间不是并列，而是流水线

```text
ChatGPT：把模糊目标变成清晰策略。
Codex：把清晰策略变成代码、文档和测试。
业务大模型：把产品输入变成职业诊断输出。
Eval/后台评分：把 AI 输出变成可检查、可沉淀的数据。
我本人：把所有 AI 输出变成最终决策。
```

## 2. ChatGPT -> Codex 交接格式

```yaml
objective: 本轮要解决什么问题
context:
  - 项目目录
  - 已有功能
  - 当前阶段
constraints:
  - 不能破坏主链路
  - 不能新增大型依赖
  - 不能夸大真实能力
tasks:
  P0:
    - 必须生成的文件
  P1:
    - 可选页面或接口
  P2:
    - 后续增强
outputs:
  - 文件清单
  - 测试结果
  - 未完成事项
```

## 3. Codex -> ChatGPT 交接格式

```yaml
changed_files:
  - path: ...
    reason: ...
validation:
  - command: ...
    result: ...
risks:
  - ...
answerable_for_ppt:
  - ...
unfinished:
  - ...
```

## 4. 业务大模型 -> Eval 交接格式

```yaml
input:
  resume_summary: ...
  target_role: ...
  evidence_status: ...
  retrieved_slices: ...
output:
  question: ...
  report: ...
  learning_tasks: ...
metadata:
  used_slice_ids: []
  evidence_constraints: []
quality_check:
  no_fabrication: true
  gap_focused: true
```

## 5. 答辩话术

```text
我不是把所有任务交给一个 AI，而是设计了分层交接协议。ChatGPT 负责上游策略，Codex 负责工程落地，业务大模型负责产品运行时生成，Eval 和人工评分负责校验，最后由我本人总控验收。
```
