# 13 评委追问：AI 协同版

## Q1：你说自己是 OPC，不就是用了 ChatGPT 和 Codex 吗？

```text
区别在于我不是临时问 AI，而是把 AI 组织成固定工作流。ChatGPT 负责赛事理解、任务拆解和 Prompt 设计；Codex 负责读取项目记忆后执行代码、脚本、测试和文档生成；业务大模型负责产品内部的简历诊断、岗位匹配和面试追问；后台评分和 Eval 负责校验输出；我本人负责目标、边界和验收。
```

## Q2：你的 AI 之间怎么交接？

```text
我用结构化任务包交接。ChatGPT 把赛事要求和项目状态转成 Codex 任务包，里面包含目标、路径、约束、输出文件和验收标准。Codex 执行后输出新增文件、修改点、测试结果和未完成事项。这些再进入任务记忆文档，形成下一轮工作的上下文。
```

## Q3：你的 Prompt 有什么技术含量？

```text
我的 Prompt 是分层设计的：战略 Prompt 用来分析赛道和评委关注点；工程 Prompt 用来约束 Codex 按项目记忆、文件路径和测试标准执行；业务 Prompt 用来约束产品内大模型必须基于简历证据和岗位画像生成；评估 Prompt 用来检查是否编造经历、是否围绕能力缺口、是否可用于报告。
```

## Q4：一个人怎么维护这么多事情？

```text
普通团队靠多人分工：产品、前端、后端、运维、职业顾问、面试官、数据标注员。我的 OPC 模式是把这些职能拆成 AI 工作流：ChatGPT 做策略，Codex 做工程，业务大模型做诊断生成，Eval 做校验，我做总控。因此一个人可以形成轻量化创新单元，替代小团队的早期研发和服务验证能力。
```
