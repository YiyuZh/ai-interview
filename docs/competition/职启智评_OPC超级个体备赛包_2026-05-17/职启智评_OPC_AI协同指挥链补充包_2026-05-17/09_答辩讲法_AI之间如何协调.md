# 09 答辩讲法：AI 之间如何协调

## 30 秒版本

```text
我的项目不是简单调用一个大模型，而是我作为 OPC 总控者，组织多个 AI 分工协作。ChatGPT 负责赛事理解、战略拆解和 Prompt 设计；Codex 负责代码、脚本、测试、部署和文档落地；业务大模型负责产品内的简历解析、岗位匹配、面试追问和报告生成；后台 Eval 与人工评分负责质量校验。我本人负责目标设定、边界控制、验收和最终决策。
```

## 60 秒版本

```text
当我拿到比赛要求后，我先使用 ChatGPT 分析赛道、评委关注点和参赛叙事；然后让 ChatGPT 把战略结论转成 Codex 可执行任务包；Codex 再根据 PROJECT_MEMORY、任务记录文档和项目约束去修改代码、生成脚本、生成材料并运行测试。在产品运行时，业务大模型负责简历解析、岗位画像理解、证据追问、报告和学习任务生成；后台人工评分和 Eval Preview 负责检查 AI 输出质量。最后由我本人做总控和责任承担。
```

## 评委问：你先怎么用 AI，后怎么用 AI？

```text
先用 ChatGPT 做赛事和策略分析，明确评委看什么；再用 ChatGPT 把分析转成 Codex 任务包；然后用 Codex 读取项目记忆并执行代码、文档和展示资产生成；产品运行时调用业务大模型生成简历诊断、面试追问和报告；最后用后台人工评分和 Eval 对 AI 输出进行校验。
```

## 评委问：Prompt 有什么设计？

```text
我的 Prompt 分四类：战略 Prompt、工程 Prompt、业务 Prompt、评估 Prompt。战略 Prompt 用于赛道和项目定位，工程 Prompt 用于 Codex 按路径和验收标准执行，业务 Prompt 用于产品内大模型基于简历证据和岗位画像生成，评估 Prompt 用于检查是否编造经历、是否围绕能力缺口、是否可用于报告。
```

## 评委问：Codex 具体做了什么？

```text
Codex 会读取任务记录和项目约束，定位 FastAPI、Vue、PostgreSQL、Docker、Caddy 等真实落点；再生成或修改后端服务、前端页面、Alembic 迁移、脚本、测试和 Markdown；最后输出验证结果和未完成事项。我再把这些工程结果转化成答辩材料和演示链路。
```
