# 评委问答库 OPC 版

## Q1：你的项目是不是只是一个 AI 面试平台？

答：

```text
不是。我现在的参赛重点不是“做了一个 AI 面试功能”，而是作为 OPC 超级个体，构建了一套高校就业服务的 AI 原生工作流。AI 面试只是其中一环，前面有简历证据、岗位画像和能力差距，后面有简历润色、学习任务、报告复盘、人工评分和样本沉淀。
```

## Q2：你作为一个人，和普通团队相比优势在哪里？

答：

```text
我的优势是轻量和快速迭代。我把传统团队中的产品、研发、就业顾问、测试、材料整理等工作拆成 AI Agent 工作流。Codex 负责工程实现，ChatGPT 负责赛事研究和材料生成，业务大模型负责就业诊断和报告组织，我本人负责目标、边界、审核和最终责任。
```

## Q3：人和 AI 的边界是什么？

答：

```text
AI 负责高频、重复、生成、检索和初步判断；人负责目标设定、资料真实性、隐私授权、评分标准、异常判断和最终决策。尤其在简历证据、岗位知识和用户数据上，系统明确不让 AI 把岗位知识写成候选人真实经历。
```

## Q4：如果现场给你一个新岗位，你怎么快速构建工作流？

答：

```text
我会先让岗位画像 Agent 提取该岗位能力模型，再让简历证据 Agent 对候选人经历做证据分层，然后 Gap Agent 输出待验证能力，面试官 Agent 生成追问，学习任务 Agent 输出补强路线，最后 Eval Agent 检查追问质量和证据约束。整个过程可以复用现有 Career-AgentOS 模板。
```

## Q5：你的垂直行业嵌入在哪里？

答：

```text
项目嵌入的是高校就业服务和大学生求职训练场景。它不是泛用聊天工具，而是围绕大学生求职准备中的岗位认知、简历证据、面试训练、学习任务和复盘反馈构建的连续工作流。
```

## Q6：你有没有真实落地？

答：

```text
当前已经有可运行原型，包含用户端、后台端、后端、数据库和部署链路；同时已经形成三岗位演示沙盘、Agent Trace、Eval Preview 和 SFT Preview。真实大规模样本和微调结果会在三个月补实路线中继续推进。
```

## Q7：你是不是已经微调了模型？

稳妥答法：

```text
当前已经完成 SFT-ready 数据结构、样本准入规则、Preview 和启动脚本。真实 OpenAI SFT 必须等真实授权样本、人工复核、验证集和官方 job 完成后，才会说完成微调。现阶段我会把它表述为后训练准备链路，而不是虚构已完成训练。
```

## Q8：为什么这符合 OPC，而不是普通创业项目？

答：

```text
OPC 考察的是 AI 工作流构建、人机协同结构和轻量化创新单元。我的项目正是把一个人通过 AI 扩展为一个就业服务小团队：我负责决策和治理，AI Agent 负责工程、材料、诊断、面试、报告和评估，从而形成一个可运行、可迭代的轻量服务单元。
```

## Q9：如果 AI 出错怎么办？

答：

```text
系统从三个层面控制风险。第一，证据约束，不把岗位知识当成候选人经历。第二，数据治理，真实样本进入评测前需要授权和去标识化。第三，人工评分，后台对问题质量、报告可执行性和学习任务可执行性进行复核。
```

## Q10：你三个月后能补实什么？

答：

```text
第一，完成 C1/C2/C3 三岗位真实闭环；第二，补足真实授权样本和后台人工评分；第三，启动 OpenAI SFT 或做 Prompt/RAG 对照 Eval；第四，小范围对接高校就业服务场景；第五，输出升级版 AI 原生结构说明、场景对接方案和实践计划。
```
