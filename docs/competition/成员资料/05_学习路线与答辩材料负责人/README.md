# 05 学习路线与答辩材料负责人

## 负责范围

- 学习路线阶段、学习任务质量、任务验收方式。
- PPT、讲稿、项目说明书、答辩问答、演示流程。
- 将真实验收结果同步到材料中，避免把未上线功能写成已完成。

## 资料应该放什么

- Python、Java、前端、产品、运营、HR 等岗位的学习路线草稿。
- 学习任务质量抽查记录：任务是否具体、是否有材料、是否有练习、是否有产出物、是否有验收方式。
- PPT 页面草稿、讲稿、答辩问答、项目说明书补充材料。
- 基于真实案例的演示故事线。

## 合格资料标准

- 学习路线必须能回答：学什么、怎么练、产出什么、怎样算学会。
- 答辩材料只能基于已实现或已验收能力书写。
- 未跑通的功能必须标记为“待验收”或“待补数据”。
