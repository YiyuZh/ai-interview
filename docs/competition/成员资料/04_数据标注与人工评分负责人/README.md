# 04 数据标注与人工评分负责人

## 负责范围

- R1-R5 真实案例台账。
- 简历证据摘录和能力项映射。
- 面试问答过程整理。
- 后台案例标注字段填写。
- AI 分数与人工评分对比。
- 重复运行记录和数据集清单。

## 样本类型判定

- 完整面试闭环样本：必须包含简历、能力诊断、至少 3-5 轮面试问答、报告、训练复盘和人工评分。
- 静态简历诊断样本：只有简历、岗位和能力诊断，尚未包含面试问答，不能作为完整真实案例。
- 重复运行记录：同一案例在相同或相近条件下重复运行，用于观察评分和报告波动。

## 资料应该放什么

- 每个案例的编号、简历来源、目标岗位、跑测日期、模型、环境、系统版本。
- 简历中能支持岗位能力的原文证据摘录。
- 能力项映射，例如沟通表达、数据分析、项目经历、岗位知识、工具使用。
- 每轮面试问答：系统问题、验证能力、用户回答、AI 评价、人工评价、是否有效验证能力。
- 报告结论和训练复盘：优势、短板、学习建议、下一轮目标。
- 人工评分记录：总分、证据对齐分、问题质量分、报告可执行性分、学习任务可执行性分。
- 至少 1 个案例重复运行 3 次的评分波动记录。

## 合格资料标准

- 不只写“好/不好”，必须写评分依据。
- AI 与人工分差异必须写明原因。
- 所有案例要能追溯到简历来源、目标岗位、系统版本和跑测环境。
- 只有简历没有问答的资料，必须标记为“静态简历诊断样本”，不能标记为完整真实案例。
- 完整案例至少 3 轮问答，推荐 5 轮问答；低于 3 轮只能作为不完整样本暂存。
