# 01 技术全栈与验收负责人

## 负责范围

- 前端、后端、后台、数据库、服务器、GitHub、Docker、Caddy、日志和部署。
- 主流程验收：用户端、后台端、健康接口、岗位画像、学习任务、模拟面试、报告、训练复盘。
- 失败问题归档：接口报错、服务器日志、截图、复现步骤、修复状态。

## 资料应该放什么

- 服务器部署命令记录。
- Alembic 迁移结果。
- Docker Compose 状态截图或日志摘要。
- 页面验收记录。
- 接口错误、Traceback、数据库缺表/缺字段错误。
- 修复后复测结果。

## 合格资料标准

- 能说明“什么时候、在哪个环境、执行了什么、结果是什么”。
- 错误资料必须包含第一条真实异常，不只写“服务器内部错误”。
- 页面验收资料必须写清访问地址、账号角色、操作步骤和通过/失败结果。
