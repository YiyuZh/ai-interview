# 资料提交说明

本说明用于规范五位维护成员提交资料的方式，目标是让后续检查和入库不混乱。

## 文件放置流程

1. 先把原始资料、摘要或链接说明放到本人角色目录的 `待检查/`。
2. 同步填写该角色目录下的 `资料登记表.csv`。
3. 审核后再移动到 `已检查可入库/`、`需修改/` 或 `不采用/`。
4. 只有 `已检查可入库/` 中的资料，后续才允许整理进数据库或正式文档。

## 文件命名建议

建议格式：

```text
岗位_主题_来源平台_日期_提交人_状态.md
```

示例：

```text
Python后端_Redis缓存一致性_牛客_2026-05-11_角色2_待核验.md
产品助理_PRD场景题_招聘JD归纳_2026-05-11_角色3_可入库.md
R1_简历证据摘录_产品助理_2026-05-11_角色4_可入库.md
```

## 必填信息

每份资料至少写清：

- 资料类型：`完整闭环样本 / 静态诊断样本 / 重复运行记录 / 岗位资料 / 问答经验 / 学习路线 / 答辩材料 / 服务器验收记录`。
- 来源链接或来源说明。
- 访问日期。
- 适用岗位。
- 对应能力项。
- 审核状态：`可入库 / 仅参考 / 待核验 / 不采用`。
- 建议入库位置：`岗位要求 / 问答经验 / 能力模型 / 面试追问 / 学习路线 / 案例标注 / 答辩材料`。
- 风险备注：是否来源不清、是否可能过时、是否只能做方向参考。

## 大文件限制

不要把大型 PDF、视频、压缩包或批量截图直接放进本目录。确实需要引用时，只放链接、摘要和页码/时间点说明。

## 完整案例口径

- 完整真实案例必须包含：简历、能力诊断、3-5 轮面试问答、报告、训练复盘和人工评分。
- 只有简历和能力诊断的资料，只能登记为 `静态诊断样本`。
- 面试问答记录至少写清：系统问题、验证能力、用户回答、AI 评价、人工评价、是否有效验证能力。
