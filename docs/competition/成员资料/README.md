# 职启智评五人维护资料区

本目录用于存放五位维护成员搜集、整理和标注的资料。所有资料先进入对应角色的 `待检查/`，经人工审核后再移动到 `已检查可入库/`、`需修改/` 或 `不采用/`。

## 使用原则

- 先登记，再入库：每份资料都要在对应角色的 `资料登记表.csv` 中记录来源、岗位、能力标签、审核状态和建议入库位置。
- 先审核，再写库：只有状态为 `可入库` 的资料，后续才允许整理进岗位画像、问答经验、学习路线、案例标注或答辩材料。
- 不放大型文件：不要放视频、压缩包、大体积数据集或未压缩截图包；大型资料只放链接、摘要和来源说明。
- 不复制来源不明内容：公开面经、招聘 JD、开源资料可以做方向归纳，但必须标注来源、访问日期、许可或风险备注。

## 五个角色目录

| 目录 | 负责内容 |
|---|---|
| `01_技术全栈与验收负责人` | 服务器、部署、接口、日志、页面验收和故障记录 |
| `02_技术岗岗位资料与面经负责人` | Python、Java、前端、测试、算法、数据分析等技术岗资料 |
| `03_非技术岗岗位资料与面经负责人` | 产品、运营、新媒体、HR、招聘、行政等非技术岗资料 |
| `04_数据标注与人工评分负责人` | R1-R5 案例、简历证据、人工评分、AI/人工对比和重复运行记录 |
| `05_学习路线与答辩材料负责人` | 学习路线、学习任务质量、PPT、讲稿、项目说明和答辩材料 |

## 审核状态

- `可入库`：来源清楚、内容可改写使用、适合进入正式知识库或数据集。
- `仅参考`：可用于理解方向，但不直接进入正式题库或训练样本。
- `待核验`：来源、时效、适用岗位或许可仍需确认。
- `不采用`：来源不明、疑似搬运、内容过时或与项目方向不匹配。

## 资料类型

登记表中的 `资料类型` 建议统一使用以下枚举，方便后续汇总和入库预检：

- `完整闭环样本`：包含简历、能力诊断、3-5 轮问答、报告、训练复盘和人工评分。
- `静态诊断样本`：只有简历和能力诊断，尚未包含面试问答，不能作为完整真实案例。
- `重复运行记录`：同一案例重复运行，用于观察评分、问题和报告波动。
- `岗位资料`：岗位要求、硬性要求、加分项、公司侧重点、典型任务。
- `问答经验`：真实面试问题、参考回答要点、复盘反思和来源审核。
- `学习路线`：学习阶段、任务质量抽查、材料和练习建议。
- `答辩材料`：PPT、讲稿、项目说明、答辩问答和展示流程。
- `服务器验收记录`：部署、迁移、日志、接口、页面验收和故障记录。
