# 04 人机协同分工图

## 1. 分工原则

```text
AI 做高频、重复、生成、检索、代码执行和初步评估。
人做目标、真实性、授权、评分标准、异常取舍和最终责任。
```

## 2. 分工表

| 环节 | 人负责 | AI 负责 | 项目证据 |
|---|---|---|---|
| 赛道选择 | 判断 OPC 与项目是否匹配 | 搜索资料、提炼评审重点 | OPC 备赛包、答辩材料 |
| 产品设计 | 定高校就业场景和功能优先级 | 生成方案、对比路径 | PROJECT_MEMORY |
| 工程实现 | 审查结果、决定是否提交 | Codex 修改代码、跑测试 | Git 提交、测试输出 |
| 简历诊断 | 确认不能编造经历 | 抽取证据、生成评分 | Resume Evidence Agent |
| 岗位画像 | 判断岗位知识边界 | RAG 切片与召回 | Role Profile RAG Agent |
| 简历润色 | 确认可改但不造假 | 生成岗位化表达建议 | Resume Polish Agent |
| 面试训练 | 判断追问是否合理 | 根据缺口生成追问 | Interviewer Agent |
| 报告复盘 | 判断建议是否可执行 | 汇总回答、缺口和任务 | Report Review Agent |
| 数据治理 | 授权、脱敏、人工复核 | PII 扫描、准入检查 | Data Governance Agent |
| 后训练 | 决定是否真实付费训练 | 生成 JSONL、dry-run | SFT Dataset Agent |

## 3. 一个人替代小团队

| 小团队角色 | OPC 替代方式 |
|---|---|
| 产品经理 | 我 + ChatGPT 策略 Agent |
| 前端工程师 | Codex Engineering Agent |
| 后端工程师 | Codex Engineering Agent |
| 运维工程师 | Codex + Docker 部署脚本 |
| 就业指导老师 | 岗位画像 + 面试官 Agent + 人工复核 |
| 简历顾问 | Resume Polish Agent + 人工边界审核 |
| 测试工程师 | Eval Judge Agent + pytest/build |
| 答辩教练 | Defense Coach Agent |

## 4. 答辩表达

```text
我的创新不是让 AI 替我完成一切，而是把原来需要产品、研发、就业老师、简历顾问、测试和运营共同完成的工作，拆成可编排、可追踪、可复盘的 AI 工作流。我作为 OPC 负责目标、标准、边界和最终责任，AI 负责高频执行和内容生产。
```
