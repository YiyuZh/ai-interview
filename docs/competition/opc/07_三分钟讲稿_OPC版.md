# 07 三分钟讲稿 OPC 版

各位评委好，我的项目叫“职启智评 OPC”。它不是传统意义上的 AI 面试网站，而是我作为一个 OPC 超级个体，围绕高校就业服务场景设计的一套 AI 原生工作流。

大学生求职中常见的问题是：不了解岗位真实能力要求，简历经历不能转化成岗位证据，模拟面试训练不连续，反馈也难以形成下一步学习任务。传统就业指导依赖老师面对面咨询，质量高但覆盖频次有限。

我的解决方案是把就业指导拆成多个 AI Agent：简历证据 Agent 判断能力是否有证据，岗位画像 Agent 提供岗位要求和面经切片，Gap Agent 分析能力差距，简历润色 Agent 在不造假的前提下优化表达，面试官 Agent 围绕缺口追问，学习任务 Agent 生成后续训练计划，Eval Agent 和数据治理 Agent 负责评分、授权和样本沉淀。

在这个结构中，AI 负责高频文本处理、检索、生成、代码执行和初步评估；我作为 OPC 负责人负责目标选择、资料真实性、隐私授权、评分标准和最终判断。通过 ChatGPT、Codex 和业务大模型 API，我一个人可以完成原本需要产品、研发、就业顾问、测试和材料团队共同完成的工作。

目前项目已经形成 Vue 用户端、Vue 管理端、FastAPI 后端、PostgreSQL、Redis/Celery 和 Docker 部署原型，具备简历解析、能力诊断、简历润色、学习任务、AI 面试、报告复盘、后台岗位画像和案例标注能力。同时，系统已经有三岗位 Agent Trace、Eval Preview 和 SFT Preview，用于展示可追溯的工作流和后训练门禁。

我不会把 Preview 说成真实训练结果。下一步三个月，我会补齐真实授权案例、人工评分、OpenAI SFT 或对照 Eval，并做高校就业服务小范围试点。职启智评的核心价值不是一个单点 AI 工具，而是一个人 + AI Agents 构建的高校就业服务轻量化创新单元。
