# 02 OPC 参赛主叙事

## 1. 一句话

```text
职启智评 OPC 是一个人 + AI Agents 构建的高校就业能力诊断与训练服务单元。
```

## 2. 三层叙事

第一层：OPC 主体能力。

```text
我作为一个人负责目标选择、任务拆解、隐私边界、真实性判断、验收标准和答辩。
```

第二层：AI 协同工作流。

```text
ChatGPT 做赛道分析和材料策略，Codex 做工程实现和测试，业务大模型做简历/岗位/面试/报告生成，Eval 和后台评分做质量校验。
```

第三层：高校就业应用。

```text
系统把大学生求职准备拆成简历证据、岗位画像、能力差距、简历润色、面试追问、学习任务、报告复盘和数据沉淀。
```

## 3. 六句话答辩主线

1. 我不是只做了一个 AI 面试网站，而是把高校就业指导拆成一套 AI 原生工作流。
2. 这套工作流由我作为 OPC 总控，多个 AI Agent 分担赛事研究、工程实现、岗位建模、简历诊断、面试追问、报告复盘和答辩材料生成。
3. AI 负责高频文本处理、检索、生成、代码执行和初步评估；人负责目标选择、真实性、隐私授权、评分标准和最终决策。
4. 系统嵌入大学生求职训练场景，解决岗位认知不清、简历证据不足、面试训练零散和反馈难复盘的问题。
5. 项目采用轻量化技术栈和外部大模型 API，不依赖大团队和重算力，一个人即可构建、部署、演示和持续迭代。
6. 后续三个月将把 Preview 链路补实为真实授权样本、人工评分、OpenAI SFT 或对照 Eval 和高校场景试点。

## 4. 最核心的四个词

- 工作流：不是一次 AI 问答，而是连续就业诊断流程。
- 协同边界：AI 做执行，人做判断和责任。
- 垂直嵌入：不是通用聊天，而是高校就业服务。
- 轻量治理：一个人可维护、可部署、可迭代、可审计。

## 5. 与原项目的关系

原项目是可运行产品底座，OPC 改造是在它之上增加：

- OPC 总控视角。
- AI 协同顺序和交接协议。
- 复赛限时实测训练包。
- OPC 答辩叙事、问答和证据材料。
