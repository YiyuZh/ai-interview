# Drill 03：60 分钟垂直场景落地方案

## 题目

请把职启智评 OPC 从“高校就业服务”迁移到一个相邻场景，例如“创新创业项目路演训练”或“研究生导师面试训练”，并说明 60 分钟内如何形成落地方案。

## 限时

60 分钟。

## 输入材料

- 新场景说明。
- 目标用户。
- 典型输入材料。
- 期望输出。

## OPC 拆解

1. 定义用户和核心任务。
2. 复用 Career-AgentOS 中的证据链、角色画像、缺口诊断、追问、报告和学习任务。
3. 替换岗位画像为新场景画像。
4. 重新定义 Eval 维度。
5. 标注哪些数据可以沉淀，哪些必须授权。
6. 给出三个月试点路线。

## AI Agent 分工

- Competition Research Agent：分析新场景需求。
- Architecture Review Agent：判断哪些模块复用、哪些模块改造。
- Codex Engineering Agent：给出最小代码或配置改造点。
- Eval Judge Agent：定义新评分维度。
- Data Governance Agent：给出数据授权和脱敏口径。

## 最终交付物

- 新场景一页方案。
- 工作流复用/改造表。
- 三个月试点计划。
- 风险边界。

## 评分点

- 是否能迁移而不是重做。
- 是否有垂直场景理解。
- 是否保留人机边界。
- 是否有试点和补实路径。

## 现场讲解话术

```text
我的系统底层不是固定的“面试题生成器”，而是一套证据链、画像、缺口、追问、报告、任务和治理的工作流模板。因此迁移到相邻场景时，主要替换画像和 Eval 维度，核心协同结构可以复用。
```
