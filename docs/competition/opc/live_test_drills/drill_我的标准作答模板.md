# 我的标准作答模板

## 1. 场景判断

```text
这个题目本质上是一个 [场景] 中的 [能力诊断/训练/复盘] 问题。它适合 OPC，因为流程可以拆成高频 AI 执行和人工边界审核。
```

## 2. 工作流拆解

```text
输入材料 -> 场景画像 -> 证据识别 -> 缺口诊断 -> 表达优化 -> 追问训练 -> 报告复盘 -> 学习任务 -> 数据治理 -> Eval/后训练准备
```

## 3. 人机协同

```text
我负责目标、真实性、授权、标准和最终责任；AI Agent 负责资料整理、检索、生成、代码执行和初步评估。
```

## 4. 交付物

```text
一张工作流图、一张 Agent 分工表、一个样例输出、一组评分标准和一个三个月补实路线。
```

## 5. 风险边界

```text
不编造用户经历，不把岗位知识当真实证据，不使用未授权数据，不把 Preview 说成真实训练结果。
```

## 6. 总结

```text
这体现的不是单点 AI 功能，而是一个 OPC 超级个体如何把多 AI 工具组织成可运行、可验收、可迭代的轻量化创新单元。
```
