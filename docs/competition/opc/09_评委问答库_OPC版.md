# 09 评委问答库 OPC 版

## Q1：这是不是只是 AI 面试平台？

不是。AI 面试只是工作流中的一环。职启智评 OPC 的重点是我作为一个人，用多 AI Agent 构建高校就业服务流程：前面有简历证据、岗位画像和能力差距，后面有简历润色、学习任务、报告复盘、人工评分和样本沉淀。

## Q2：OPC 体现在哪里？

OPC 体现在一个人负责目标、边界、验收和最终责任，同时用 ChatGPT、Codex、业务大模型和 Eval 扩展产品、工程、内容、测试和答辩能力。项目不是靠大团队堆人，而是靠 AI 工作流组织能力。

## Q3：人和 AI 的边界是什么？

AI 做高频执行：生成、检索、代码、初步诊断、报告组织。人做最终判断：场景选择、资料真实性、隐私授权、评分标准、异常处理和提交决策。

## Q4：如果现场给一个新岗位，怎么快速适配？

先由岗位画像 Agent 提取能力模型，再由简历证据 Agent 做证据分层，Gap Agent 输出待验证能力，Interviewer Agent 生成追问，Learning Plan Agent 生成补强任务，Eval Judge Agent 检查输出质量和证据约束。

## Q5：你是不是已经微调模型？

没有这样夸大。当前完成的是 SFT-ready 数据结构、样本准入、Preview 和启动脚本。真实 OpenAI SFT 必须等真实授权样本、人工复核、验证集和官方 job 完成后才能宣称完成。

## Q6：如果 AI 乱编怎么办？

系统用三层控制：证据约束、数据治理、人工复核。岗位知识库不能写成候选人经历，真实样本进入评测或训练前必须授权、去标识化、人工评分并通过 PII 扫描。

## Q7：为什么适合高校就业服务？

高校就业服务有大量重复但需要质量控制的工作：简历诊断、岗位认知、模拟面试、学习建议和复盘。它适合 AI 提升覆盖频次，同时保留老师或人工复核的判断权。

## Q8：三个月能补实什么？

补 C1/C2/C3 真实闭环，收集授权且人工复核的样本，形成 holdout eval，启动 OpenAI SFT 或 Prompt/RAG 对照评估，并进行小范围就业服务试点。
