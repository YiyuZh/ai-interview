# 06 答辩 PPT OPC 版页面结构

## 1. 推荐 12 页

1. 我是谁：OPC 超级个体，而不是传统团队。
2. 我解决什么问题：高校就业服务覆盖不足、反馈难闭环。
3. 我的 AI 原生工作流：从简历到复盘再到样本沉淀。
4. 我的人机协同结构：人控目标和边界，AI 做执行。
5. OPC 如何替代小团队：产品、研发、就业顾问、测试、运营。
6. Career-AgentOS 多 Agent 架构：13+ Agent 分层。
7. 可运行系统底座：用户端、后台、FastAPI、PostgreSQL、Redis/Celery、Docker。
8. 三岗位 Agent Trace：Python 后端、产品助理、人力资源专员。
9. Eval Preview 与 SFT-ready：展示评估方法和训练门禁，不冒充真实训练。
10. 复赛限时实测能力：现场如何构建新岗位工作流。
11. 三个月补实路线：真实授权样本、人工评分、OpenAI SFT、场景试点。
12. 总结：一个人 + AI Agents 的高校就业能力诊断与训练服务单元。

## 2. 主标题

```text
职启智评 OPC：一个人 + AI Agents 的高校就业能力诊断与训练服务单元
```

副标题：

```text
基于 Career-AgentOS 的 AI 原生工作流、人机协同结构与轻量化创新实践
```

## 3. 必须保留的边界页脚

```text
当前展示为 Career-AgentOS Preview 与 SFT-ready 工程门禁；不宣称真实 OpenAI SFT 已完成，不把 demo 样本当作真实用户数据。
```

## 4. 不建议使用的标题

- “AI 面试官系统”
- “已完成大模型微调”
- “真实训练效果提升”
- “真实用户数据集”
