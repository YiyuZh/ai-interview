# 01 赛事适配判断

## 1. 赛事理解

OPC 超级个体赛道不是单纯考察“做了什么软件”，而是考察一个青年创新主体如何在 AI 环境中构建轻量化创新单元。评委会关注：

- AI 工作流构建能力。
- 人机协同逻辑设计。
- 产业应用场景表达。
- 一个人如何借助 AI 扩展产品、工程、运营和答辩能力。
- 集中限时实测中能否快速把新任务拆成可执行工作流。

## 2. 职启智评适配优势

职启智评已经有真实工程底座，不是纯文档项目：

- 前后端、后台、数据库、异步任务和 Docker 部署链路存在。
- 已形成高校就业服务垂直场景。
- 业务闭环覆盖简历、岗位、诊断、润色、面试、报告、学习任务、复盘和人工评分。
- Career-AgentOS 已具备三岗位 demo、Trace、Eval Preview 和 SFT Preview。
- 项目长期通过任务记忆文档、Codex 提交、测试报告和部署脚本沉淀 AI 协同证据。

## 3. 主要短板

- 原叙事容易被理解成普通 AI 面试产品。
- 需要更明确展示“我如何作为 OPC 总控 AI Agents”。
- 复赛集中限时实测材料需要单独训练。
- 真实样本和真实微调仍待补实，不能作为当前已完成成果宣传。

## 4. 推荐参赛口径

```text
我作为 OPC 超级个体，围绕高校就业服务场景，设计并运行 Career-AgentOS 多 Agent 工作流。系统把就业咨询小团队的工作拆成可编排 AI Agent：简历证据、岗位画像、能力差距、简历润色、面试追问、学习任务、报告复盘、数据治理、Eval 和 SFT-ready 样本准备。AI 负责高频执行，人负责目标、真实性、授权、标准和最终责任。
```

## 5. 赛道建议

主赛道表达：

```text
人工智能与新一代信息技术 / 数字经济与企业服务 / 高校就业服务数字化
```

答辩重点不放在“面试题生成是否准确”一个点上，而放在“一个人如何用 AI 构建、运营、校验并持续迭代一个就业服务单元”。
