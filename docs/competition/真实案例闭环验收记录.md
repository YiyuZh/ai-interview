# 真实案例闭环验收记录

本文件用于记录服务器真实运行后的闭环验收结果。它不是演示稿，也不是预设结论；只有在系统真实跑完后，才把结果填入这里。

## 1. 使用规则

- 每个案例必须有明确来源：用户授权简历、公开授权简历或团队自造合成案例。
- 不写虚构分数。系统没跑出来的字段统一写“待跑测”。
- 不把一次成功当作稳定结论。关键案例至少重复运行 3 次。
- 技术岗和非技术岗都要覆盖，不能只用一个 Python 后端案例证明全部能力。
- 人工评分至少 2 人独立打分，再记录平均分和差异原因。

## 1.1 CSV 记录工具

追加一条标准记录：

```powershell
cd D:\apps\ai-interview
python scripts\append_real_closed_loop_record.py --case-id R1 --set resume_source=2590603008詹已誉简历.pdf --set target_position=产品助理/产品经理实习生
```

指定某次跑测结果：

```powershell
python scripts\append_real_closed_loop_record.py --case-id R1 --set upload_status=成功 --set ability_diagnosis_status=成功 --set interview_start_status=成功 --set report_status=成功 --set training_review_status=成功 --set admin_interview_record=成功 --set evaluation_dataset_status=空态 --set status=部分通过
```

检查 CSV：

```powershell
python scripts\validate_real_closed_loop_records.py
```

真实案例补齐后的严格验收：

```powershell
python scripts\validate_real_closed_loop_records.py --strict
```

严格验收默认要求：至少 5 个案例、5 条完整闭环记录、5 条已完成人工评分记录、至少 1 个案例完成 3 次重复运行，并且没有待跑测行和失败行。

说明：脚本会自动给同一案例递增 `run_id`。字段名必须来自 `datasets/真实闭环验收记录模板.csv` 表头，写错字段会报错。
