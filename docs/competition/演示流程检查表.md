# 演示流程检查表

## 演示前 30 分钟

- [ ] 后端服务已启动，健康接口可访问。
- [ ] 用户端页面可访问。
- [ ] 后台端页面可访问。
- [ ] DeepSeek 或 OpenAI API Key 已保存并测试成功。
- [ ] 至少准备 1 份 PDF 简历样例。
- [ ] 公共岗位画像已初始化。
- [ ] 浏览器已登录用户端和后台端。
- [ ] 参赛演示页 `/competition-demo` 能正常打开。

## 用户端演示

- [ ] 打开工作台，说明项目定位。
- [ ] 打开参赛演示页，展示岗位画像、评分规则、人工评分对比。
- [ ] 上传简历，先用“Python后端开发工程师”或其他技术岗跑一遍。
- [ ] 再用“产品助理 / 人力资源专员 / 运营助理”等非技术岗跑一遍。
- [ ] 展示简历解析和岗位匹配报告。
- [ ] 选择公共岗位画像。
- [ ] 开始模拟面试，回答至少 2 道题。
- [ ] 展示追问闭环和证据核查信息。
- [ ] 打开求职能力评估报告。

## 后台演示

- [ ] 打开公共岗位画像库，展示技术岗与非技术岗默认画像。
- [ ] 打开测评记录，展示用户测评历史。
- [ ] 打开测评详情，展示人工标注区域。
- [ ] 打开评测样本与人工评分对比，展示可导出数据。

## 答辩重点

- [ ] 说明不是套壳 ChatGPT，而是多岗位画像、轻量匹配评分、知识切片、评分规则和证据约束的闭环系统。
- [ ] 说明没有自研大模型，也没有做真实微调。
- [ ] 说明当前验证方式是测试案例、人工评分对比和输出稳定性检查。
- [ ] 说明后续可扩展轻量评分校准模型。

## 兜底方案

- [ ] 如果模型调用失败，演示参赛演示页和后台样本数据。
- [ ] 如果简历解析卡住，展示历史测评记录和报告。
- [ ] 如果云服务器不可用，使用本地前后端演示。
