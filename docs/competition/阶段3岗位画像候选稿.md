## 1. 使用方法

审核时按下面四步检查：

1. 看岗位是否该保留：是否适合大学生求职和本项目演示。
2. 看画像是否完整：核心能力、典型任务、关键词、企业考察方向、样例题是否齐全。
3. 看是否能进入知识库：内容是否能被切成“岗位要求、知识点、项目追问、场景追问、评分点”等片段。
4. 看是否有风险：是否整段复制公开资料，是否夸大为真实企业内部题库，是否把技术岗排除。

审核结论可以写在每个岗位的“人工审核栏”。

## 2. 合格标准

每个岗位至少满足：

- 核心能力不少于 6 项。
- 典型任务不少于 5 项。
- 关键词不少于 8 个。
- 企业常见考察方向不少于 3 条。
- 样例面试题不少于 5 个。
- 技术岗必须覆盖八股知识点、项目追问和排障思路。
- 非技术岗必须覆盖场景任务、沟通协作、流程意识、数据复盘和结果表达。
- 资料来源只做归纳改写，不复制整段 JD、面经或开源资料。

## 3. 资料来源原则

本候选稿综合公开资料进行归纳，后续写入系统时只保留改写后的岗位画像和题目方向。

### 5.1 Python后端开发工程师

岗位定位：负责后端接口、业务逻辑、数据库、缓存、异步任务和服务部署，适合计算机、软件工程、人工智能相关专业学生演示。

核心能力：

- Python 语言基础和异常处理
- FastAPI / Flask / Django 等 Web 框架
- SQL、事务、索引和数据库建模
- Redis 缓存、过期策略和基础数据结构
- HTTP、RESTful API、认证鉴权
- 异步编程、任务队列和并发基础
- Docker、Linux、日志和基础部署
- 项目问题定位和接口排障

典型任务：

- 开发业务接口和后台管理接口
- 设计数据库表结构和查询逻辑
- 接入缓存、异步任务和消息通知
- 编写接口文档和基础单元测试
- 排查接口超时、数据库慢查询和线上异常
- 配合前端完成接口联调

关键词：

- Python
- FastAPI
- SQLAlchemy
- PostgreSQL
- MySQL
- Redis
- Docker
- Linux
- HTTP
- RESTful
- Celery
- 异步
- 索引
- 事务

企业常见考察方向：

- 是否能讲清一个后端项目的接口设计、数据库设计和权限控制。
- 是否理解数据库索引、事务、慢查询和缓存的基本取舍。
- 是否具备排障意识，例如日志定位、复现路径、接口状态码和异常处理。
- 是否能把“我参与了项目”说成个人职责、技术方案、结果指标和复盘。

样例面试题：

1. 你做过的后端项目中，最复杂的接口是什么？请说明入参、鉴权、数据库操作和异常处理。
2. PostgreSQL 或 MySQL 中索引为什么能提升查询速度？什么情况下索引可能失效？
3. Redis 在你的项目里解决了什么问题？如果缓存和数据库不一致，你会怎么处理？
4. FastAPI 的依赖注入或中间件你用过吗？适合解决什么问题？
5. 一个接口偶发 500 或超时，你会按什么顺序排查？
6. 如果简历只写“负责后端开发”，你认为还需要补充哪些证据？

评分关注点：

- 能否把技术名词和项目证据绑定。
- 能否解释数据库、缓存和接口设计的取舍。
- 能否给出清晰的排障路径。

知识库切片建议：

- Python 语言基础
- Web 框架与接口设计
- 数据库事务与索引
- Redis 缓存
- 异步任务
- Docker 与部署
- 项目追问
- 线上排障
