# 八分钟讲稿

## 1. 项目定位
职启智评面向大学生求职准备，解决岗位认知不清、简历证据不足、面试训练泛化和学习路径不具体的问题。

## 2. 核心闭环
系统主链路是：上传简历、解析简历、选择目标岗位、能力诊断、证据约束简历润色、模拟面试、报告生成、学习任务、训练复盘、后台人工评分。

## 3. Career-AgentOS 架构
我们把系统拆成 13 个角色，包括 OPC 总控、简历证据、岗位画像、能力差距、简历润色、面试追问、报告、学习任务、数据治理、Eval、后训练路线和评委拷问等 Agent。每个 Agent 有固定输入、输出和边界。

## 4. 简历润色 Agent
润色不是替学生编经历。已证明能力可以强化表达，间接证明能力可以补清场景和动作，仅声明或缺失能力只能提示补证据。

## 5. 演示闭环
比赛版新增三岗位 demo cases：Python 后端、产品助理、人力资源。每个案例都能导出 Agent Trace，展示证据链、岗位画像、能力缺口、润色建议、追问、报告摘要、Eval 和 SFT Preview。

## 6. Eval 与 SFT Preview
Eval 使用七维 35 分规则评分，展示多 Agent 输出相对 `baseline_prompt_preview` 规则基线的沙盘差异；这不是实际模型调用对比。SFT Preview 只验证数据结构和训练任务设计，不等同真实微调。

## 7. 数据治理
真实训练必须满足用户授权、去标识化、人工复核和样本准入。演示样本明确标记 `demo_constructed` 和 `for_training=false`。

## 8. 后续路线
赛后三个月补齐真实授权样本，完成 OpenAI SFT、holdout eval、DPO 偏好数据和 QLoRA 私有化验证。
