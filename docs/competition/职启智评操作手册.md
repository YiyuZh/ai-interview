# 职启智评操作手册

适用项目：`D:\apps\ai-interview`

作品题目：职启智评：基于大语言模型与岗位画像匹配的大学生求职能力评估及模拟面试系统

本手册是 `docs/competition/` 目录的总入口。当前路线已纠偏：项目主系统继续保留原有多岗位 AI 模拟面试生态，`职启智评` 是参赛品牌和增强层，不把系统收窄成人力资源岗位项目。

- “大学生求职能力评估”只作为参赛增强页面、报告区块和评分维度，不覆盖全站主流程。

## 4. 推荐推进路线

### 快速可展示路线

1. 启动后端、用户端、后台端。
2. 执行公共岗位画像初始化脚本。
3. 用技术岗样例跑一遍：Python 后端 / 前端开发 / 测试工程师任选其一。
4. 用非技术岗样例跑一遍：产品助理 / 人力资源专员 / 运营助理任选其一。
5. 在简历分析页展示关键词覆盖率、语义分、规则分、最终匹配分。
6. 在 `/competition-demo` 展示多岗位画像、技术深度路线和 AI/人工评分对比。

过关效果：老师能看到项目不是单岗位问答，也不是纯 ChatGPT 套壳，而是有岗位画像、知识切片、量化匹配和证据约束。

### 标准参赛路线

1. 按 `职启智评项目升级流程手册.md` 从阶段 0 推进到阶段 12。
2. 技术岗和非技术岗都至少准备 3 个岗位画像。
3. 每个案例记录：关键词覆盖率、规则分、语义分、LLM 评分、最终分、人工分。
4. 后台保留至少 5 条可导出的评测样本。
5. 形成项目说明书、答辩问答、测试对比表、演示流程、Word 手册和 PPT。

过关效果：系统、数据、材料、答辩逻辑互相支撑。

### 技术加分路线

1. 先用 TF-IDF/余弦相似度完成可复现语义分。
2. 再接入 embedding API，和 TF-IDF 分数做对比。
3. 基于人工评分表调整规则权重，形成轻量评分校准。
4. 导出 JSONL 样本，为未来微调评分模型做准备。
5. 不在样本不足时夸大“真实微调”。

过关效果：答辩能讲清机器学习、深度学习 API、证据约束和微调准备各自做了什么。
