# 职启智评技术深度升级路线

适用项目：`D:\apps\ai-interview`

本文件专门说明“机器学习、深度学习、模型微调准备”怎样真实融入当前项目。核心原则：能做什么说什么，不用术语虚构能力。

## 1. 总体定位

项目主线是多岗位 AI 模拟面试平台：

```text
简历解析 -> 岗位画像匹配 -> 模拟面试 -> 回答评价 -> 报告生成
```

参赛增强层是：

```text
岗位画像库
+ 岗位知识切片
+ 关键词覆盖率
+ TF-IDF/余弦相似度
+ 规则评分
+ LLM/embedding 语义能力
+ 人工评分对比
+ fine-tune-ready JSONL
```

## 2. 机器学习层

### 数据从哪里来

机器学习层不需要一开始就下载大规模数据集。当前项目先用三类小数据闭环：

| 数据 | 来源 | 用途 |
|---|---|---|
| 岗位画像与题库 | 公开职业资料、公开面试资料、人工整理 | 生成岗位关键词、知识点和题目 |
| 简历-JD 匹配样例 | 公开合成数据、匿名样例、项目自产样例 | 验证关键词覆盖率和语义相似度 |
| AI-人工评分样本 | 后台人工标注和评测样本导出 | 调整规则权重，准备后续微调 |

详细建设步骤见：`数据集与岗位知识库建设手册.md`。

### 当前可落地能力

| 能力 | 做法 | 输出 |
|---|---|---|
| 关键词覆盖率 | 简历文本与岗位画像关键词做命中统计 | `keyword_coverage` |
| TF-IDF/余弦相似度 | 不依赖重库，计算简历与画像的文本相似度 | `semantic_score` |
| 规则评分 | 组合关键词、语义分、证据强度、LLM 参考分 | `rule_score`、`final_score` |
| 评分解释 | 输出命中词、缺口词、证据不足点 | `evidence_basis` |

### 答辩说法

> 本项目不是完全把评分交给大模型自由发挥，而是先构建岗位画像和评分规则，再使用关键词覆盖率、TF-IDF/余弦相似度和规则权重形成可复核的匹配指标。大模型主要负责结构化理解、追问生成和解释性报告。

## 3. 深度学习应用层

### 当前能力

- 使用大语言模型 API 做简历解析、岗位分析、面试出题、回答评价和报告生成。
- Prompt 要求结构化 JSON 输出。
- 通过证据约束降低编造：没有简历证据时只能提示“不足”或“需要追问”。

### 后续可增强

| 能力 | 做法 | 验收 |
|---|---|---|
| Embedding 语义匹配 | 简历和岗位画像转向量，计算 cosine similarity | 有 `embedding_status` 和向量相似度 |
| TF-IDF fallback | embedding API 不可用时继续返回 TF-IDF 分 | 服务不崩溃 |
| Prompt 版本对比 | V1/V2/V3 固定样例重复测试 | 记录分数波动 |

### 答辩说法

> 大语言模型本身属于深度学习自然语言处理模型。项目没有从零训练底层模型，而是通过岗位画像、结构化 Prompt、证据约束和语义匹配，把通用模型能力转化为高校就业指导场景的可解释应用。

## 4. 微调准备层

当前不直接宣称真实微调。正确路线是先做数据闭环：

1. 保存简历解析结果。
2. 保存岗位画像和知识切片。
3. 保存面试问题、回答、AI 评分和反馈。
4. 后台人工审核并给出人工评分。
5. 导出 JSONL，形成 fine-tune-ready 数据。
6. 离线比较 Prompt 版本、模型版本、规则权重。
7. 样本足够后再做评分模型或小模型微调试点。

### JSONL 推荐结构

```json
{
  "target_position": "Python后端开发工程师",
  "resume_summary": "候选人简历摘要",
  "position_profile": "岗位画像摘要",
  "question": "面试问题",
  "answer": "候选人回答",
  "ai_score": 7.8,
  "human_score": 8.0,
  "rubric": "评分依据",
  "accepted": true
}
```

### 答辩说法

> 当前阶段微调处于数据准备和离线评测阶段。我们已经设计了人工评分、样本导出和 JSONL 数据结构，避免在样本不足时直接宣称完成微调。

## 5. 岗位知识库路线

岗位知识库不是下载完数据集就结束，而是把“岗位会考什么”整理成面试官可用的结构化资料。

技术岗必须包含：

- Python / Java / JavaScript 基础。
- 数据库、索引、事务。
- Redis、缓存一致性。
- HTTP、网络、浏览器。
- 并发、异步、线程。
- 算法与数据结构。
- 系统设计。
- 项目追问。
- 线上排障。

非技术岗必须包含：

- 典型任务。
- 工作流程。
- 场景问题。
- 数据复盘。
- 沟通协作。
- 简历证据追问。

## 6. 验收指标

| 类别 | 过关标准 |
|---|---|
| 数据源清单 | 有岗位画像、面试题库、匹配样例、AI-人工评分四类数据说明 |
| 岗位题库 | 每个重点岗位至少 5 个样例问题，并说明企业常见考察方向 |
| 多岗位 | 技术岗与非技术岗都能跑通主流程 |
| 量化匹配 | 简历分析包含关键词覆盖率、语义分、规则分、最终分 |
| 证据约束 | 缺证据时会追问，不编造 |
| 后台验证 | 能保存人工评分和导出样本 |
| 稳定性 | 同一案例重复 3 次有记录 |
| 微调准备 | 有 JSONL 结构和样本导出路线 |

## 7. 不该做的事

- 不临时引入重型向量数据库。
- 不做爬虫。
- 不做视频/语音面试。
- 不把所有页面改成人力资源岗位。
- 不说“已完成大模型微调”，除非真的有数据、训练、对比实验和复现记录。
