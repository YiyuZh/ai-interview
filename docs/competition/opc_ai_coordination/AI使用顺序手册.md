# AI 使用顺序手册

## 1. 标准顺序

1. ChatGPT：理解赛事、生成策略、拆任务。
2. Codex：读取仓库、落地代码/文档、跑测试、提交。
3. 业务大模型：在产品运行时处理简历、岗位、追问、报告。
4. Eval/后台评分：检查质量、授权、PII 和样本准入。
5. 我本人：决定是否采纳、提交、部署和答辩。

## 2. 为什么这样排序

- ChatGPT 适合开放分析和表达重构。
- Codex 适合工程落地和仓库级修改。
- 业务大模型适合产品内生成任务。
- Eval/后台评分适合质量闭环。
- 人负责最终判断和责任。

## 3. 每轮任务模板

```text
ChatGPT 输出：目标、约束、任务包、验收标准。
Codex 输入：任务包 + 仓库文件 + 记忆文档。
Codex 输出：代码/文档/脚本 + 测试结果 + 提交。
业务大模型输出：候选诊断、追问、报告。
Eval 输出：评分、风险、是否准入。
人类输出：最终验收和下一步。
```
