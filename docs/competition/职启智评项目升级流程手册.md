## 2026-05-10 初赛后评委反馈纠偏

初赛后评委的核心反馈已经整理到 `docs/competition/评委点评整理_正式版.md`。后续升级不能继续只把项目讲成“AI 模拟面试工具”，而要升级为：

```text
就业能力诊断与提升平台：
简历解析 -> 目标岗位 -> 岗位能力图谱 -> 能力差距诊断 -> 个性化学习路径 -> 模拟面试训练 -> 进度复盘
```

新的项目主线如下：

- `模拟面试` 继续保留，但它只是能力提升闭环中的训练模块，不再是唯一核心。
- `岗位画像` 升级为 `岗位能力图谱 + 岗位知识库`，既要描述岗位需要什么能力，也要支撑问题生成、差距诊断和学习路径。
- `能力提升系统` 成为赛后重点建设方向：系统不仅指出用户哪里不足，还要根据目标岗位、当前能力和求职时间，生成可执行的学习计划、任务、练习和进度记录。
- `AI` 的角色改为辅助解析、生成、解释和反馈；项目自身必须有岗位能力结构、差距评分模型、知识库分区、学习进度机制和人工复核数据。

### 赛后核心升级原则

- 不删除原有多岗位生态。技术岗、产品岗、运营岗、HR 岗、行政岗等都继续支持。
- 不把所有页面改成聊天机器人。后续要逐步形成独立页面：岗位能力图谱、个人能力画像、能力差距报告、学习路径、训练记录。
- 不泛泛写“AI 建议”。所有建议必须能追溯到岗位要求、简历证据、能力差距或面试表现。
- 不急着做重数据库迁移。第一步优先用现有 `knowledge_content`、`focus_points`、`interviewer_prompt` 承载结构化 Markdown 分区，稳定后再评估是否新增表。

### 岗位知识库新分区

后续公共岗位画像和用户私有岗位画像都按下面结构组织：

| 分区 | 内容 | 用途 |
|---|---|---|
| 岗位要求 | 硬性要求、加分项、公司侧重点、岗位常见 JD 表述 | 生成岗位能力图谱和匹配标准 |
| 问答经验 | 真实面试问题、参考回答、回答反思、易踩坑点 | 支撑模拟面试题和复盘 |
| 能力模型 | 能力项、等级标准、权重、可提升系数 | 支撑能力差距量化评分 |
| 面试追问 | 证据追问规则、评分关注点、回答改进建议 | 支撑面试训练闭环 |

## 0. 总路线纠偏

当前项目不是从零做一个“人力资源岗位测评小工具”，而是在原 `ai-interview` 多岗位模拟面试系统上做参赛增强。

正确结构：

```text
原系统主线：用户设定岗位 -> 选择岗位画像/知识库 -> 开始模拟面试 -> 生成报告
    ↓
参赛增强：岗位画像库 + 轻量匹配评分 + 证据追问 + 人工评分对比 + 微调数据准备
```

因此后续所有阶段都必须遵守：

- 保留技术岗，例如 Python 后端、Java 后端、前端开发、测试工程师、算法工程师、数据分析师。
- 保留非技术岗，例如产品助理、人力资源专员、招聘助理、运营助理、行政管理助理、新媒体运营。
- 不把“大学生求职能力评估”覆盖成全站唯一语义，它是参赛增强功能、报告维度和展示页面。
- 技术深度不靠堆词，而要落到可运行指标、样本、对比和导出数据。

提升优先级 = 岗位重要性 × 能力差距 × 可提升系数
