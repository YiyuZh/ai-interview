## 2. 下一步推荐路线

### 2.1 运行验收

目标：确认服务器上的真实系统可以跑，而不是只在代码层通过。

要做：

1. 部署最新代码。
2. 执行数据库迁移。
3. 执行公共岗位画像种子脚本。
4. 检查用户端、后台端、健康接口。
5. 检查公共岗位画像、测评样本、面试记录页面。

过关后进入：真实简历主流程验收。

### 2.2 真实案例闭环

目标：用一份真实简历验证系统主线。

要做：

1. 上传真实简历。
2. 选择目标岗位。
3. 查看能力诊断。
4. 把缺失能力加入学习任务。
5. 开始模拟面试。
6. 生成报告。
7. 进入训练复盘。

过关后进入：真实样本和人工评分沉淀。

### 2.3 数据沉淀

目标：让项目不只是页面能跑，还能拿出验证数据。

要做：

1. 准备 5-10 个真实或公开授权案例。
2. 每个案例记录目标岗位、简历证据、能力差距、学习任务、面试表现、报告结论。
3. 人工评分至少覆盖技术岗和非技术岗。
4. 同一案例重复运行 3 次，记录评分波动。

过关后进入：是否做服务器持久化和材料更新。

### 2.4 服务器持久化评估

目标：判断学习任务和训练复盘是否从浏览器本地存储升级为数据库。

当前默认：先保持本地存储。

升级条件：

1. 用户确实需要跨设备同步。
2. 学习任务和复盘记录已经被验证有价值。
3. 已确定数据库表、接口、权限和迁移方案。

### 2.5 材料更新

目标：让项目说明、答辩材料、演示脚本与当前系统一致。

要做：

1. 更新项目说明书。
2. 更新答辩问答。
3. 更新演示流程检查表。
4. 更新测试案例与人工评分对比。
5. 必要时重新生成 PPT、Word、表格。

### 阶段 6：Prompt 与证据约束

目标：让简历分析、面试追问和报告都基于证据，不编造经历。

当前状态：已有基础。

过关标准：

- 简历没写的技能不能直接判定为已掌握。
- 报告能指出证据不足。

### 阶段 8：Embedding 可选增强

目标：在不破坏 TF-IDF 回退的前提下增加语义向量匹配。

- 配置有效 DeepSeek 或 OpenAI API Key。

- 上传简历触发分析。

过关标准：

下一阶段：阶段 17。

### 阶段 17：岗位能力图谱与能力差距模型

目标：让系统能解释用户与目标岗位的能力差距从哪里来。

当前状态：实施初版，待人工审核和服务器验收。

操作步骤：

- 检查岗位能力项。
- 检查等级、权重、关键词、证据线索、可提升系数。
- 用真实简历查看能力差距输出。

过关标准：

- 每个重点岗位至少有 6 个能力项。
- 能力差距能对应简历证据。
- 优先级不是空泛排序。

不通过处理：能力项空泛时先人工修画像；差距缺失时查 `ability_gap_profile`。

下一阶段：阶段 18。

### 阶段 18：岗位知识库结构化改造

目标：把岗位画像升级为可维护的结构化岗位知识库。

当前状态：实施初版，待人工审核和运行验收。

操作步骤：

- 后台编辑公共岗位画像。
- 检查四分区：岗位要求、问答经验、能力模型、面试追问。
- 保存后重建切片。
- 查看切片分类。

过关标准：

- 四分区内容能保存和回填。
- 旧格式内容不会丢失。
- 切片能识别不同分区。

不通过处理：保存失败查后台接口；切片分类失败查 Markdown 标题。

下一阶段：阶段 19。

### 阶段 19：能力提升系统与学习进度

目标：把能力差距转成可执行学习任务，并记录进度。

当前状态：学习任务中心已实施初版，待真实任务质量验收。

操作步骤：

- 从报告或能力诊断中把缺失能力加入学习任务。
- 打开 `/learning-tasks`。
- 勾选完成、填写笔记、记录薄弱项变化。
- 导出和导入学习任务 JSON。

过关标准：

- 任务有能力项、目标岗位、来源报告或简历。
- 任务能执行，不只是泛泛建议。
- 刷新后进度仍保留。
- 导入导出可恢复。

不通过处理：任务太空泛时回到学习计划生成逻辑；本地丢失时查 `zhiqizhice:learning-tasks:v1`。

下一阶段：阶段 20。

### 阶段 20：用户端能力诊断页面

目标：让用户独立查看能力差距、证据说明和提升入口。

当前状态：实施初版，待真实简历验收。

操作步骤：

- 打开 `/ability-diagnosis`。
- 指定或自动读取最近完成简历。
- 查看匹配分、能力差距、证据说明。
- 把缺失项加入学习任务。

过关标准：

- 没有结构化差距时显示“暂无结构化能力差距数据”，不能误写“没有明显差距”。
- 有证据缺口时能生成兜底差距卡片。
- 文案使用“当前简历 / 这份简历 / 当前用户”，避免像招聘裁决。

不通过处理：低匹配简历显示无差距时，优先修前端兜底逻辑和后端 `ability_gap_profile`。

下一阶段：阶段 21 或阶段 22。

### 阶段 21：后台岗位知识库维护页面

目标：让后台能真实维护岗位知识资产。

当前状态：实施初版，待人工审核和后台运行验收。

操作步骤：

- 打开后台公共岗位画像。
- 新建或编辑画像。
- 填写四个分区。
- 保存并重建切片。

过关标准：

- 后台名称为“职启智评管理后台”，不再使用“参赛后台管理”等比赛口径。
- 页面风格简洁，黑灰白为主。
- 四分区完整度、问答经验数量、来源审核状态能提示维护质量。

不通过处理：发现比赛话术继续残留时统一改成真实项目口径。

下一阶段：阶段 22。

### 阶段 22：训练闭环与复盘报告

目标：把能力诊断、学习任务、模拟面试和报告串成持续训练闭环。

当前状态：实施初版，待真实训练闭环验收。

操作步骤：

- 完成一次模拟面试。
- 打开 `/training-review`。
- 选择已完成面试。
- 查看能力差距、学习任务完成情况、面试结果、下一轮建议。
- 填写自评、复盘笔记、下一轮目标。

过关标准：

- 无已完成面试时有清晰空状态。
- 有已完成面试时能生成复盘摘要。
- 能读取学习任务完成数和未完成任务。
- 复盘笔记刷新后仍保留。

不通过处理：无面试记录先查 `/interviews`；报告读取失败查 `getReport`；复盘丢失查 `zhiqizhice:training-review:v1`。

下一阶段：回到阶段 11/12 做真实数据沉淀，或评估服务器持久化。

## 4. 服务器验收命令

服务器部署：

```bash
cd /opt/apps/ai-interview
git fetch origin
git reset --hard origin/main
docker compose up -d --build app frontend admin
docker compose restart app frontend admin
```

数据库迁移：

```bash
docker compose exec app alembic current
docker compose exec app alembic heads
docker compose exec app alembic upgrade head
```

公共岗位画像种子：

```bash
docker compose exec app python -m app.scripts.seed_competition_knowledge_bases
```

数据库检查：

```bash
docker compose exec postgres sh -lc 'psql -U "$POSTGRES_USER" -d "$POSTGRES_DB" -c "select scope, count(*) from position_knowledge_bases group by scope;"'
docker compose exec postgres sh -lc 'psql -U "$POSTGRES_USER" -d "$POSTGRES_DB" -c "select count(*) as slice_count from position_knowledge_base_slices;"'
```

日志检查：

```bash
docker compose logs --tail=300 app
docker compose logs --tail=300 app | grep -Ei "Traceback|UndefinedColumn|relation .* does not exist|interviews/start|evaluation-datasets"
```

Redis/Celery 配置检查：

```bash
grep -E "^(REDIS_HOST|REDIS_PORT|REDIS_PASSWORD|CELERY_BROKER_URL|CELERY_RESULT_BACKEND)=" .env
```

建议：`REDIS_HOST=redis`、`REDIS_PORT=6379`。如果手写的 `CELERY_BROKER_URL` 把密码误放到端口位置，应先修 `.env`。

## 5. 人工验收清单

用户端：

- 能登录。
- 能上传简历。
- 能选择或输入目标岗位。
- 能查看能力诊断。
- 能把缺失能力加入学习任务。
- 能开始模拟面试。
- 能生成报告。
- 能进入训练复盘。

后台端：

- 公共岗位画像数量不少于 12。
- 能编辑四分区知识库。
- 能保存并重建切片。
- 面试记录能看到新建记录。
- 测评样本无数据时显示空态，不显示红色 500。

真实案例：

- 至少 1 个技术岗案例。
- 至少 1 个非技术岗案例。
- 每个案例记录简历证据、目标岗位、能力差距、学习任务、面试表现、报告结论。
- 后续扩展到 5-10 个案例。

材料：

- 项目说明书与当前系统一致。
- PPT 不写内部边界话术。
- 答辩问答能说明：系统不是简单调用 AI，而是岗位知识库、能力模型、差距诊断、学习任务和训练复盘的闭环。
