# 微调与后训练路线：赛前展示到三个月补实

## 1. 技术选择总览

结合当前项目状态，推荐路线是：

```text
赛前：Agent 编排 + RAG + Eval Preview + SFT Preview
第 1 个月：真实授权样本 + OpenAI SFT
第 2 个月：偏好样本 + DPO
第 3 个月：开源模型 + QLoRA 私有化降本验证
```

---

## 2. 为什么第一阶段仍然主打 SFT

SFT 适合解决：

```text
让模型学习标准输出格式。
让模型稳定按照职启智评的业务口径生成追问。
让模型围绕能力缺口、证据状态和岗位画像输出。
```

第一版 SFT 任务：

```text
根据去标识化简历摘要、目标岗位、能力缺口、证据状态、岗位画像切片和面试上下文，生成下一轮高质量面试追问。
```

不建议第一版训练完整报告，因为输出太长、变量太多、样本量要求高。

---

## 3. SFT 样本格式

OpenAI chat fine-tuning 风格示例：

```json
{
  "messages": [
    {
      "role": "system",
      "content": "你是职启智评的证据追问型 AI 面试官。必须围绕岗位能力缺口和简历证据状态追问，不能编造候选人经历。"
    },
    {
      "role": "user",
      "content": "目标岗位：Python 后端开发工程师\n能力缺口：Redis 缓存设计\n证据状态：缺失\n简历摘要：候选人做过 Flask 接口项目，但未出现 Redis 或缓存优化经历。\n面试上下文：第一轮已介绍项目背景。请生成下一轮追问。"

    {

  }
```

## 4. DPO 二阶段路线

DPO 适合解决：

```text
模型不只是学标准答案，还要知道什么追问更好。
```

DPO 数据格式思想：

```text
同一个输入
chosen：高质量追问
rejected：泛泛追问或编造经历的追问
```

示例：

```json
{
  "prompt": "目标岗位：Python 后端；能力缺口：Redis；证据状态：缺失；请生成追问。",
  "chosen": "你简历中未体现 Redis 相关证据，请结合一个具体项目说明是否处理过缓存穿透、击穿或一致性问题，当时采取了什么方案，结果如何？",
  "rejected": "请介绍一下你对 Redis 的理解。",
  "preference_reason": "chosen 更贴合证据缺口，并要求具体项目和结果指标。"
}
```

DPO 的比赛表达：

```text
我们已设计偏好对齐数据结构，后续会把人工评分中的高质量追问和低质量追问成对沉淀，形成 DPO 偏好训练集。
```

---

## 5. QLoRA 三阶段路线

QLoRA 适合解决：

```text
长期降低 API 成本。
探索私有化部署。
在开源模型上做低成本参数高效微调。
```

QLoRA 不适合作为当前 API-only 项目的第一主线，但非常适合放到三个月补实路线中。

建议开源模型候选：

```text
Qwen 系列中文模型
Llama 系列模型
DeepSeek / Yi 等支持中文和指令能力的开源模型
```

比赛表达：

```text
当前阶段采用 OpenAI-compatible API 保证效果和开发速度，后续将用 QLoRA 在开源模型上复现实验，以降低调用成本并提升可控性。
```

## 6. 赛前可以展示什么

赛前不一定要真的完成全部训练，但要展示“技术设计完成 + 工程入口完成”：

```text
1. SFT 任务定义。
2. SFT Preview JSONL。
3. Eval 评分维度。
4. DPO chosen/rejected 数据结构。
5. QLoRA 私有化路线图。
6. 三个月补实计划。
```

PPT 表述：

```text
我们已经完成面向就业诊断任务的后训练技术路线设计，并在工程侧打通 SFT-ready 数据准备与 Eval 入口。赛前版本主要展示多 Agent 编排和 SFT Preview，后续三个月将按真实授权样本逐步完成 OpenAI SFT、DPO 偏好对齐和 QLoRA 私有化验证。
```

## 7. 三阶段验收标准

### 阶段 1：SFT

```text
真实授权样本 >= 10
总训练样本 30-50
验证样本 >= 5
OpenAI job 成功
拿到 fine_tuned_model
完成 holdout eval
```

### 阶段 2：DPO

```text
至少 50 对 chosen/rejected 偏好样本
每对样本有人工评分原因
完成偏好评估
```

### 阶段 3：QLoRA

```text
选定一个开源基座模型
完成数据转换
完成单卡或云 GPU LoRA/QLoRA 训练
完成 base vs qlora 对比
输出成本对比表
```
