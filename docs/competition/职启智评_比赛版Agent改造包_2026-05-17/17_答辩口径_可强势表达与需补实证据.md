# 答辩口径：可强势表达与需补实证据

本文件用于平衡“比赛表达效果”和“三个月后可补实”。

---

## 1. 可以强势表达的内容

这些是基于项目设计、工程链路和演示资产可以表达的：

```text
1. 已完成 Career-AgentOS 多智能体就业诊断架构设计。
2. 已完成从简历证据、岗位画像、能力缺口到面试追问的业务链路设计。
3. 已完成面试追问生成任务的 SFT-ready 数据结构设计。
4. 已完成 Eval Judge 六维评分体系设计。
5. 已完成三岗位沙盘案例和 Agent Trace 展示流程。
6. 已完成 SFT、DPO、QLoRA 三阶段后训练路线设计。
7. 已完成一人 OPC 架构下的虚拟 Agent 分工和 Codex 推进任务清单。
```

---

## 2. 可以包装得更有技术感的表达

普通说法：

```text
我们做了一个 AI 面试官。
```

升级说法：

```text
我们构建了一个面向就业能力诊断的 Career-AgentOS 多智能体系统。
```

普通说法：

```text
我们让大模型根据简历生成问题。
```

升级说法：

```text
我们通过岗位画像 RAG、简历证据链和能力缺口矩阵约束大模型生成证据追问。
```

普通说法：

```text
我们以后准备微调。
```

升级说法：

```text
我们已经完成面向面试追问任务的后训练工程路线设计，包含 SFT-ready 样本、Eval 评分、DPO 偏好样本和 QLoRA 私有化降本路径。
```

---

## 3. 关于“微调完成”的推荐表达

如果赛前还没有真实 job，不要说：

```text
已经完成真实 OpenAI 微调并拿到 fine_tuned_model。
```

可以说：

```text
我们已经完成面向面试追问生成任务的微调方案设计、样本结构、SFT Preview 和训练入口封装。
```

更比赛化一点：

```text
赛前版本已经完成后训练闭环的工程化准备和演示级验证，三个月内按真实授权样本完成 OpenAI SFT、DPO 和 QLoRA 补实。
```

如果三个月后真实完成，再改为：

```text
已完成一次 OpenAI SFT 实验，并保存 fine_tuned_model、job id、样本统计和 holdout eval 报告。
```

---

## 4. 关于“效果提升”的推荐表达

如果只有沙盘评估，不要写固定真实提升百分比。

推荐说法：

```text
在三岗位沙盘评估中，Career-AgentOS 相比普通 Prompt 在能力缺口聚焦、证据约束和追问深度上表现更稳定。
```

PPT 可展示：

```text
base_prompt：17/30
agent_optimized：29/30
```

但标题写：

```text
沙盘评估 / Preview Eval
```

如果后续真实 holdout eval 完成，再写真实指标。

---

## 5. 关于 DPO 和 QLoRA 的推荐表达

可以说：

```text
我们已完成 DPO 偏好对齐数据结构设计，后续会把人工评分中的 chosen/rejected 追问对沉淀为偏好样本。
```

可以说：

```text
我们已规划 QLoRA 私有化降本路线，后续将在开源模型上复现面试追问任务，降低长期 API 成本。
```

不要说：

```text
已经完成不存在的 DPO 训练。
已经完成不存在的 QLoRA adapter。
```

---

## 6. 关于真实数据的推荐表达

可以说：

```text
系统设计了用户授权、去标识化、后台人工评分和样本准入机制。
```

可以说：

```text
赛前演示使用三岗位沙盘案例，后续将逐步替换和补充真实授权样本。
```

不要说：

```text
演示样本全部来自真实用户。
```

