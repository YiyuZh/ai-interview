版本日期：2026-05-17

项目定位：从“AI 面试官”升级为“多智能体就业能力诊断与训练平台”

---

## 1. 本包的目标

本包不是单纯微调路线，而是面向比赛答辩和三个月后项目审查的**适应性改造方案**。

核心目标：

```text
先让项目在 PPT 答辩中显得有体系、有创新、有工程闭环；
同时所有设计都能在三个月内被真实代码、真实样本、真实评估逐步补实。
```

推荐主叙事：

```text
职启智评不是一个简单调用大模型的 AI 面试官，
而是一个基于 Career-AgentOS 的多智能体就业能力诊断与提升平台。
系统将简历证据、岗位画像、能力缺口、证据约束简历润色、模拟面试、学习任务、人工评分、Eval 与微调数据闭环串成一条可持续优化的 AI 工程链路。
```

## 3. 本包对原项目的总体改造方向

原项目已有能力：

```text
简历解析
目标岗位
岗位画像 / 岗位知识库
能力诊断
简历润色建议
学习任务
模拟面试
面试报告
训练复盘
用户授权
后台人工评分
SFT 数据准备链路
```

比赛版改造后要呈现为：

```text
Career-AgentOS 多智能体诊断引擎
+ 岗位画像 RAG Agent
+ 简历证据链 Agent
+ 能力差距 Agent
+ 简历润色 Agent
+ AI 面试官 Agent
+ 报告与学习任务 Agent
+ 数据治理 Agent
+ Eval Judge Agent
+ SFT / DPO / QLoRA 路线 Agent
+ OPC 一人指挥中心
```

一句话：

```text
原来是一个“会问问题的 AI 面试系统”；
现在要包装和改造成一个“能诊断、能追问、能评分、能沉淀样本、能持续微调优化的就业能力智能体系统”。
```

## 5. 赛前表达原则

本包采用“比赛表达优先，但不伪造不可追溯事实”的策略。

可以强势表达：

```text
已完成多智能体就业诊断架构设计。
已完成面试追问微调任务设计。
已完成 SFT-ready 数据闭环设计和 Preview 生成链路。
已完成三岗位演示沙盘与 Agent Trace 方案。
已完成面向后续 SFT / DPO / QLoRA 的三阶段技术路线设计。
```

不建议伪造的内容：

```text
不存在的 OpenAI fine_tuned_model ID。
不存在的训练 job id。
不存在的真实用户样本规模。
没有评测记录支撑的固定提升百分比。
把构造样本说成真实授权样本。
```

答辩上可以使用更有竞争力的表述：

```text
我们已经完成面向就业诊断任务的模型后训练方案设计与工程化落地入口，
当前赛前版本已形成演示级闭环，
三个月内将按真实授权样本、OpenAI SFT、偏好数据 DPO 和开源模型 QLoRA 的路线逐步补实。
```
