# 比赛答辩技术叙事与 PPT 结构

## 1. 答辩主线

```text
大学生求职准备的核心问题，不是缺少一个聊天机器人，而是缺少一套能对齐目标岗位、识别简历证据、定位能力差距、持续训练改进的诊断系统。

职启智评通过 Career-AgentOS 多智能体架构，将岗位画像、简历证据链、能力差距诊断、证据约束简历润色、模拟面试、学习任务、人工评分、Eval 与后训练数据闭环串联起来，使每一次训练都成为下一轮模型优化的数据资产。
```

---

## 2. 13 页 PPT 结构

### 第 1 页：标题

```text
职启智评：基于 Career-AgentOS 的多智能体就业能力诊断与训练平台
```

副标题：

```text
从简历证据到岗位画像，从模拟面试到学习路径，从 Eval 到模型后训练闭环
```

---

### 第 2 页：问题痛点

讲三个痛点：

```text
1. 大学生不了解目标岗位真实能力要求。
2. 简历描述很多是“自我声明”，缺少证据支撑。
3. 通用 AI 面试官容易泛泛提问，无法形成训练闭环。
```

---

### 第 3 页：产品闭环

放链路图：

```text
简历解析 -> 岗位画像 -> 简历证据评分 -> 能力差距诊断 -> 简历润色 -> 模拟面试 -> 报告生成 -> 学习任务 -> 用户授权 -> 人工评分 -> Eval/SFT 样本沉淀
```

---

### 第 4 页：核心创新 Career-AgentOS

展示多 Agent 架构图。

讲法：

```text
我们不是让一个大模型完成所有任务，而是将就业诊断拆解为多个专业 Agent，并通过统一的证据状态和岗位画像协同。
```

---

### 第 5 页：简历证据链

展示表格：

| 能力项 | 简历证据 | 证据状态 | 风险 | 追问方向 |
|---|---|---|---|---|
| REST API | Flask 项目接口 | 间接证明 | 缺少性能指标 | 接口设计与异常处理 |
| Redis | 未出现 | 缺失 | 岗位关键能力缺口 | 缓存场景与一致性 |
| MySQL | 写“熟悉” | 仅声明 | 缺少优化案例 | 慢查询与索引 |

---

### 第 6 页：岗位画像 RAG

展示：

```text
目标岗位：Python 后端开发工程师
岗位核心能力：接口设计、数据库、Redis、并发、部署、排障
系统约束：问题必须贴合岗位能力，不把岗位要求写成候选人经历
```

---

### 第 7 页：证据约束简历润色

展示：

```text
能力状态：Redis 缓存缺失
润色边界：不能写成“负责 Redis 缓存优化”
可用建议：保留接口开发和 MySQL 设计事实，同时提示补充真实缓存或性能优化证据
```

讲法：

```text
简历润色 Agent 不是替学生编经历，而是把真实经历表达得更贴岗位；缺少证据的能力只给补证据建议。
```

---

### 第 8 页：证据追问型 AI 面试官

放对比：

```text
通用 AI：请介绍一下你的后端项目。

职启智评：你简历中提到参与 Flask 接口开发，但没有看到 Redis 或性能优化证据。请结合一个具体项目说明你是否处理过接口响应慢、缓存一致性或并发请求问题，当时采取了什么方案，结果指标如何？
```

---

### 第 9 页：人类在环与数据闭环

展示：

```text
用户授权 -> 去标识化 -> 后台人工评分 -> 样本质量筛选 -> train/validation 拆分 -> Eval -> SFT/DPO 数据池
```

讲法：

```text
系统每一次高质量训练复盘，都可以沉淀成后续模型优化样本。
```

---

### 第 10 页：Eval 评估体系

展示 7 个维度：

```text
能力缺口聚焦
证据约束
追问深度
岗位贴合
润色可执行性
格式稳定
可用于报告
```

讲法：

```text
我们不只展示单次生成效果，而是用固定评分维度衡量模型是否更适合就业诊断任务。
```

---

### 第 11 页：后训练技术路线

展示三阶段：

```text
阶段 1：SFT —— 学会标准追问格式、证据约束和简历润色边界。
阶段 2：DPO —— 学会好追问和差追问的偏好差异。
阶段 3：QLoRA —— 在开源模型上低成本私有化训练，降低长期成本。
```

---

### 第 12 页：现场演示

演示流程：

```text
1. 选择 Python 后端岗位。
2. 读取演示简历。
3. 展示简历证据链。
4. 展示能力缺口矩阵。
5. 展示证据约束简历润色建议。
6. 生成证据追问。
7. 展示报告摘要。
8. 展示 Eval 和 SFT Preview。
```

---

### 第 13 页：总结与路线图

三句话：

```text
职启智评不是单点 AI 面试，而是多智能体就业诊断闭环。
我们用岗位画像、简历证据链和证据追问，让面试更贴合真实岗位。
后续通过授权样本、人工评分、Eval、SFT、DPO 和 QLoRA 持续优化系统能力。
```

## 4. 强化版答辩开场稿

```text
各位评委老师好，我们的项目叫职启智评。

它不是一个简单的 AI 面试官，而是一套面向大学生求职准备的多智能体就业能力诊断与训练平台。

传统 AI 面试往往停留在“根据简历问几个问题”，但真实求职更关键的是：目标岗位到底要求什么能力，学生简历中的能力有没有证据，缺口在哪里，应该如何追问，面试后又如何转化成学习任务和模型优化数据。

因此我们设计了 Career-AgentOS 架构，将简历证据 Agent、岗位画像 Agent、能力差距 Agent、简历润色 Agent、AI 面试官 Agent、报告 Agent、数据治理 Agent、Eval Agent 和后训练 Agent 串联起来，形成从诊断、润色、训练、评估到微调优化的闭环。
```

## 5. 结尾稿

```text
我们的创新不在于简单调用大模型，而在于把大模型能力场景化、流程化和可评估化。

通过 Career-AgentOS，职启智评可以做到三件事：
第一，基于岗位画像和简历证据发现真实能力缺口；
第二，通过证据追问型面试推动学生暴露和补强短板；
第三，通过授权样本、人工评分、Eval 和后训练路线，让系统持续变得更懂就业诊断。

这就是职启智评从 AI 面试工具升级为就业能力智能训练平台的核心价值。
```
