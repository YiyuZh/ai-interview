# 前端与后台改造规范：Vue 展示层

## 1. 改造目标

前端不需要大改。赛前只要能让评委看到：

```text
多 Agent 不是 PPT 空话，而是系统中有 trace、有证据链、有证据约束简历润色、有评分表、有 SFT Preview。
```

---

## 2. 用户端新增页面

页面名称：

```text
Career-AgentOS 技术展示
```

路径建议：

```text
/competition/agent-trace
/tech/agent-trace
```

页面结构：

```text
顶部：项目定位说明
左侧：演示案例选择
中间：Agent 工作流时间线
右侧：当前步骤输出
底部：简历润色建议 + Eval 分数 + SFT Preview 摘要
```

---

## 3. 页面模块

### 模块 1：案例选择

```text
Python 后端开发工程师
产品助理 / 产品经理实习生
人力资源专员
```

### 模块 2：Agent Timeline

```text
1. 简历证据 Agent
2. 岗位画像 Agent
3. 能力差距 Agent
4. 简历润色 Agent
5. 面试追问 Agent
6. 报告 Agent
7. Eval Judge Agent
8. SFT Preview Agent
```

### 模块 3：证据链表格

| 能力 | 简历证据 | 证据状态 | 追问方向 |
|---|---|---|---|

### 模块 4：能力缺口矩阵

| 能力 | 岗位要求 | 候选人证据 | 缺口等级 | 追问重点 |
|---|---|---|---|---|

### 模块 5：简历润色建议

展示结构化输出：

```text
总体润色策略：...
可强化表达：...
不能编造的能力：...
待补证据：...
风险提示：...
```

### 模块 6：AI 追问展示

展示结构化输出：

```text
问题：...
目标能力：...
证据状态：...
追问原因：...
期望回答元素：...
```

### 模块 7：Eval 分数

七维雷达或表格：

```text
能力缺口聚焦
证据约束
追问深度
润色可执行性
岗位贴合
格式稳定
可用于报告
```

如果不想做图，表格即可。

---

## 4. 后台新增页面

页面名称：

```text
样本闭环与后训练 Preview
```

展示：

```text
样本 ID
样本来源
授权状态
人工评分状态
Eval 状态
SFT Preview 状态
是否可进入真实训练
```

赛前可先展示 demo 数据。

---

## 5. UI 文案

页面头部文案：

```text
Career-AgentOS 将就业诊断拆解为多个专业 Agent，通过岗位画像、简历证据链和能力缺口矩阵生成证据追问，并将高质量案例沉淀为 Eval 和后训练样本。
```

SFT Preview 文案：

```text
当前展示为演示级 SFT Preview，用于验证样本结构、字段完整性和后训练链路。后续接入真实授权样本后，可用于正式 OpenAI SFT 或开源模型微调。
```

---

## 6. 不必优先做的前端功能

赛前不建议做：

```text
复杂拖拽 Agent 编排器
在线训练按钮
完整训练任务管理 UI
复杂权限系统重构
多模型路由管理后台
```

这些可以放到三个月计划。
