# 三个月补实计划

本计划用于支撑“比赛允许先完成技术设计，项目审查在三个月后”的节奏。

---

## 总目标

三个月后，把赛前的演示级架构补成真实可查的工程闭环：

```text
真实授权样本
真实 OpenAI SFT job
真实 holdout eval
真实 DPO 偏好样本
真实 QLoRA 开源模型验证
```

---

## 第 1 个月：补实 SFT

### 目标

```text
完成面试追问生成任务的真实 OpenAI SFT 实验，并评估是否把证据约束简历润色作为第二个窄任务加入训练。
```

### 任务

```text
1. 修复 Git 发布断点，确保服务器拉到最新脚本。
2. 补充 C1/C2/C3 真实闭环案例。
3. 获取真实用户授权。
4. 后台人工评分。
5. 去标识化处理。
6. 生成 train.jsonl 和 validation.jsonl。
7. 创建 OpenAI fine-tuning job。
8. 查询 job 状态，保存 job_record.json 和 job_status.json。
9. 完成 holdout eval。
```

### 验收

```text
真实授权样本 >= 10
训练样本 30-50
验证样本 >= 5
拿到 fine_tuned_model
完成 base vs fine-tuned eval，评估维度包含追问质量和润色边界控制
```

---

## 第 2 个月：补实 DPO 偏好对齐

### 目标

```text
让模型学习“什么追问更好”。
```

### 任务

```text
1. 从真实面试和构造样本中生成多版本追问。
2. 后台人工选择 chosen / rejected。
3. 记录偏好原因。
4. 构建 DPO 数据集。
5. 如果使用 OpenAI 不支持 DPO，则先作为偏好评估和开源模型训练数据。
6. 用 DPO 数据改进提示词和样本选择。
```

### 验收

```text
偏好样本对 >= 50
每对都有 chosen/rejected 和 reason
生成 dpo_dataset.preview.jsonl
完成偏好数据质量报告
```

---

## 第 3 个月：补实 QLoRA 私有化验证

### 目标

```text
验证开源模型低成本微调路线，降低长期依赖 API 的风险。
```

### 任务

```text
1. 选择中文能力较好的开源模型。
2. 转换 SFT 数据为开源训练格式。
3. 配置 LoRA/QLoRA 训练脚本。
4. 进行小规模训练。
5. 导出 adapter。
6. 完成 base vs qlora eval。
7. 输出成本对比。
```

### 验收

```text
完成一次 QLoRA 小规模实验
保存训练参数
保存 adapter 或训练输出
完成 Eval 对比
输出 API 成本 vs 私有化成本估算
```
