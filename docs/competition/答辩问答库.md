# 答辩问答库

## Q1：你们这个和直接问 ChatGPT 有什么区别？

普通 ChatGPT 是开放式问答，输出容易不稳定，也缺少统一评分依据。职启智评保留多岗位模拟面试主流程，并引入岗位画像库、知识切片、关键词覆盖率、TF-IDF 语义分、规则评分、简历证据核查和结构化报告，把简历诊断、岗位匹配、模拟面试、能力报告和人工评分对比整合成闭环流程。

## Q2：你们有没有训练自己的模型？

当前阶段没有训练底层大模型，也没有声称自研大模型或真实微调已上线。我们使用大语言模型 API 作为自然语言理解和生成能力基础，重点做应用层创新，包括岗位画像构建、轻量匹配评分、Prompt 约束、证据追问、人工评分对比和 fine-tune-ready JSONL 数据准备。

## Q3：用了哪些人工智能技术？

主要包括大语言模型文本理解与生成、简历结构化抽取、岗位画像匹配、关键词覆盖率、TF-IDF/余弦相似度、规则评分、Prompt 工程、多轮问答生成、STAR 结构识别、回答质量评价和可解释报告生成。后续会把 embedding 语义向量匹配作为可选增强接入。

## Q4：评分标准怎么来？

评分标准参考岗位胜任力思想，并结合大学生求职场景设计了岗位匹配度、专业能力、实践经历、表达逻辑、完整度和改进潜力六个维度。系统先根据岗位画像、关键词覆盖、TF-IDF 相似度和简历证据生成可解释指标，再由大语言模型给出解释性建议。

## Q5：AI 输出不稳定怎么办？

系统采用三类措施：第一，要求结构化输出；第二，通过岗位画像和评分规则限制输出范围；第三，对同一案例进行多次测试，并与人工评分进行对比。后续会继续扩大样本规模，并引入轻量评分校准机制。

## Q6：你们是不是把原来的技术面试功能删掉了？

没有。纠偏后的路线是“通用多岗位 AI 面试平台 + 职启智评参赛增强层”。技术岗仍然是重点支持对象，包括 Python 后端、Java 后端、前端开发、测试工程师、算法工程师和数据分析师。新增 HR、运营、行政等岗位是扩展岗位生态，不是替代技术岗位。

## Q7：为什么选择大学生求职场景？

团队专业背景是工商管理，与人力资源、劳动关系和就业指导场景贴合。这个选题既能体现 AI 技术应用，也能体现专业理解，适合做成可演示、可复核、可继续扩展的系统。

## Q8：项目目前最大的不足是什么？

当前样本规模仍然有限，人工评分对比还需要继续扩大；系统依赖外部大语言模型 API，稳定性受模型服务影响。后续重点是积累真实样本、完善评分校准和增强离线演示能力。
