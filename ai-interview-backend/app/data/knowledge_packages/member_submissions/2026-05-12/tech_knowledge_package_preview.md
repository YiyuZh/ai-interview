# Python后端开发工程师岗位资料包

目标岗位：Python后端开发工程师

## 岗位要求
硬性要求：本科及以上，计算机、软件工程、地理信息、测绘等相关专业优先；熟练掌握 Python，理解面向对象、数据结构与算法、操作系统、计算机网络、数据库原理；Python 方向需了解 Django、Flask、FastAPI 中至少一种框架；熟悉 Git、Docker、Linux、接口调试、数据库索引、Redis 缓存与常见服务部署。
加分项：GIS/智慧城市、支付系统、直播技术、AI/RAG 系统等垂直项目经验；熟悉高并发、分布式架构、微服务、云原生部署、自动化测试、SaaS 基础设施；有 GitHub 开源项目、技术博客或英文文档协作经验。
常见任务：业务系统开发、接口设计、数据库设计、性能与稳定性优化、缓存策略、异步任务、第三方系统或模型接口集成、线上故障定位和复盘。
公司侧重点：互联网大厂关注高并发、系统解耦和稳定性；GIS 方向关注 PostGIS、空间数据处理和地图服务；AI/RAG 后端关注文档切片、向量检索、Prompt 工程和幻觉排查；DevOps/平台可靠性方向关注 Docker/K8s、Prometheus、ELK 和服务治理。

## 问答经验

### 问答经验 1
- 真实问题：请做一个 1-2 分钟自我介绍，并介绍你做过的后端项目业务逻辑、具体工作和最大技术难点。
- 参考回答要点：按背景、架构、核心模块、技术选型、个人贡献、结果组织；技术难点要说明问题、分析过程、方案、结果和复盘。
- 复盘反思：避免只讲业务或罗列技术名词，需要补充性能提升、稳定性改善、接口响应、数据量等真实指标。
- 公司/场景：京东后端实习、大厂后端通用技术面
- 考察能力：项目表达、后端工程经验、问题复盘
- 难度：★★★
- 来源说明：成员整理：技术岗资料包，来源包含京东后端实习公开面经和多份招聘 JD 汇总，2025
- 审核状态：可入库

### 问答经验 2
- 真实问题：谈谈你对 RAG 的理解；文档入库如何分片；用户问题过长导致效果差时怎么优化；如何评估 RAG 效果？
- 参考回答要点：RAG 包括检索、增强、生成；分片可按固定长度、语义、递归分割并保留重叠窗口；长问题可做拆分、关键信息提取、上下文压缩；评估关注召回率、相关性、准确率和幻觉率。
- 复盘反思：需要区分知道概念和做过系统，追问向量库、Embedding、Rerank、检索失败排查和低置信回答处理。
- 公司/场景：AI/RAG 后端方向
- 考察能力：RAG 工程理解、检索增强、幻觉排查
- 难度：★★★★
- 来源说明：成员整理：AI/RAG 后端方向公开 JD 与面经归纳，2025
- 审核状态：可入库

### 问答经验 3
- 真实问题：如何用 Redis 实现分布式互斥锁？Redis 常见缓存问题有哪些？数据库索引设计的核心原则是什么？
- 参考回答要点：分布式锁可用 SET key value NX EX timeout，加 Lua 脚本保证原子解锁；缓存穿透可用布隆过滤器或空值缓存，击穿用互斥锁或热点 key 保护，雪崩用过期时间打散和集群；索引应关注区分度、联合索引最左前缀、覆盖索引和回表成本。
- 复盘反思：如果简历只写 Redis，需要追问是否真实处理过并发、锁超时、缓存一致性、索引失效和慢查询。
- 公司/场景：后端开发、高并发业务线
- 考察能力：Redis、数据库、并发与性能优化
- 难度：★★★★
- 来源说明：成员整理：后端公开面经与数据库/缓存高频题归纳，2025
- 审核状态：可入库

### 问答经验 4
- 真实问题：如何设计支付系统应对双十一高并发场景？如何保障服务稳定性？
- 参考回答要点：可从分库分表、读写分离、限流降级、消息队列异步处理、幂等设计、监控告警、日志、熔断、容灾、灰度发布、超时重试、补偿机制和数据对账展开。
- 复盘反思：重点不是堆术语，而是解释每个方案解决哪个风险，以及候选人在真实项目中的责任边界。
- 公司/场景：支付系统后端、大厂高并发业务线
- 考察能力：系统设计、稳定性保障、高并发处理
- 难度：★★★★★
- 来源说明：成员整理：大厂高并发业务线公开面经归纳，2025
- 审核状态：可入库

## 能力模型
能力项：Python 基础与异步编程、Web API 设计、数据库索引与事务、Redis 缓存与分布式锁、接口联调与异常处理、Docker/Linux 部署、RAG 工程理解、线上排障复盘。
等级标准：初级能说明概念和基础用法；中级能结合项目解释设计取舍；较高等级能说明线上问题、性能瓶颈、监控指标和复盘改进。
证据线索：真实项目中的接口、数据库表、缓存 key、异步任务、部署脚本、故障日志、性能指标、RAG 检索效果和排查记录。

## 面试追问
简历只写熟悉 Python/FastAPI/Redis/SQL 但没有项目证据时，不直接判定掌握，应追问使用场景、责任边界、关键参数、异常处理和排障过程。简历提到后端项目但未写具体框架时，可先按间接证据追问接口设计、数据库、缓存、部署和性能优化。RAG 类题目只用于验证工程理解，不得把岗位知识库内容当作候选人真实经历。

## 重点关注
Python/FastAPI/Django/Flask、数据库索引与事务、Redis 缓存、异步任务、Docker/Linux、RAG 工程、接口联调、线上排障。

## 面试官提示
围绕岗位必需能力追问真实项目证据。对简历声明型能力先验证，不直接相信；对间接证据追问具体场景、参数、取舍和结果；不得替候选人编造项目经历。

# Java后端开发工程师岗位资料包

目标岗位：Java后端开发工程师

## 岗位要求
硬性要求：本科及以上，计算机相关专业优先；掌握 Java 基础、面向对象、集合、多线程、JVM 基础；了解 Spring Boot、Spring Cloud、RESTful 接口、数据库事务、索引、Redis、消息队列、Linux、Git、Docker。
加分项：高并发、分布式系统、微服务、支付系统、金融政务、大型业务系统、服务降级、容灾、监控告警、灰度发布经验。
常见任务：业务模块开发、接口设计、数据库设计、服务治理、缓存和消息队列接入、性能优化、线上故障排查。
公司侧重点：大厂核心服务偏 Java/Go，金融政务大型系统更看重稳定性、事务一致性、权限、审计和可维护性。

## 问答经验

### 问答经验 1
- 真实问题：给定两组无序整型数据，找出两组数据的交集，并去除重复元素。
- 参考回答要点：可用哈希集合，遍历 A 存入集合，再遍历 B 判断并去重；也可排序后双指针遍历。需要分析时间复杂度和空间复杂度，并说明不同数据规模下的取舍。
- 复盘反思：面试中要边写边讲思路，不能只给最终答案；算法题用于验证基础逻辑和复杂度意识。
- 公司/场景：通用后端技术面、大厂基础算法题
- 考察能力：数据结构与算法、复杂度分析
- 难度：★★★
- 来源说明：成员整理：通用后端公开面经归纳，2025
- 审核状态：可入库

### 问答经验 2
- 真实问题：如何设计支付系统应对高并发和重复提交？
- 参考回答要点：结合限流、消息队列、幂等 token、唯一订单号、分布式锁、事务补偿、对账、监控告警和降级策略说明。
- 复盘反思：需要追问是否真实做过相关模块，还是只停留在八股记忆。
- 公司/场景：支付系统、金融或大流量业务
- 考察能力：高并发系统设计、幂等、稳定性
- 难度：★★★★★
- 来源说明：成员整理：大厂高并发业务线公开面经归纳，2025
- 审核状态：可入库

## 能力模型
能力项：Java 基础、Spring 生态、数据库与事务、Redis 与消息队列、分布式与高并发、服务稳定性、接口联调、日志监控与排障。
证据线索：Spring 项目模块、接口文档、数据库表设计、Redis 使用场景、MQ 消费幂等、线上日志、监控指标和性能优化记录。

## 面试追问
如果简历只写 Java/Spring/Redis，优先追问是否做过接口、事务、缓存一致性、幂等和异常补偿。项目经历不足时，不直接判会，只标记为声明待验证。

## 重点关注
Java/Spring、数据库事务、Redis、消息队列、幂等、分布式、高并发、稳定性和排障。

## 面试官提示
不要只问概念定义，要让候选人说明真实项目中的接口、表结构、并发问题、补偿机制和监控指标。

# 前端开发工程师岗位资料包

目标岗位：前端开发工程师

## 岗位要求
硬性要求：熟练 HTML、CSS、JavaScript ES6+；至少掌握 Vue 或 React；了解 MVVM、组件化、状态管理、路由、构建工具和浏览器基础；本科及以上、计算机相关专业优先；有项目、实习或开源经验更好。
加分项：跨端/H5/小程序/桌面端、页面性能优化、前端工程化、AI 辅助开发工具、前后端接口协作、长期实习稳定性。
常见任务：业务页面开发、交互实现、接口联调、性能优化、兼容性处理、线上问题修复、技术文档和组件规范沉淀。
公司侧重点：字节国际电商关注亿级用户体验和性能；百度文库关注跨端与文档产品；腾讯 AI Agent 关注跨平台应用和 AI 产品前端；传统企业关注基础 Web 项目稳定交付。

## 问答经验

### 问答经验 1
- 真实问题：如何和后端协作完成接口联调？遇到接口定义不一致时怎么处理？
- 参考回答要点：提前对齐 RESTful、字段命名、错误码和异常场景；用 mock 数据推进前端开发；发现不一致时先还原需求场景，再同步后端更新接口文档和测试用例。
- 复盘反思：避免只说配合后端，需要体现流程意识、冲突处理和文档更新。
- 公司/场景：字节跳动/通用前端协作场景
- 考察能力：接口联调、沟通协作、工程流程
- 难度：★★★
- 来源说明：成员整理：前端公开面经与 JD 归纳，2024-2025
- 审核状态：可入库

## 能力模型
能力项：HTML/CSS/JS 基础、Vue/React 组件化、接口联调、状态管理、性能优化、浏览器机制、工程化与构建、沟通协作。
证据线索：页面截图、组件代码、接口文档、联调记录、性能指标、上线问题和修复复盘。

## 面试追问
简历只写 Vue/React 时，追问组件通信、状态管理、生命周期、接口联调、跨域、性能优化和真实页面职责；TypeScript/JavaScript、Node 工程化等相关能力可作为间接证据，但仍需验证项目使用深度。

## 重点关注
JavaScript、Vue/React、组件通信、接口联调、跨域、性能优化、工程化、项目协作。

## 面试官提示
前端面试要结合真实页面、组件、接口和性能问题追问，不要只停留在 CSS/JS 八股。

# 算法工程师岗位资料包

目标岗位：算法工程师

## 岗位要求
硬性要求：本科及以上，计算机、数学、统计、自动化、电力电子等相关专业；掌握机器学习、深度学习基础，熟悉 Python，部分岗位要求 C/C++ 或 Matlab；了解常见机器学习模型、数据处理、模型评估和业务问题转化。
加分项：Kaggle/天池竞赛、NLP/多模态/大模型测评或微调经验、算法工程化部署、广告推荐/电商/通信/电力电子等垂直业务经验。
常见任务：算法建模与优化、模型评估、评估数据构建、业务落地协作、技术调研、数据分析和文档沉淀。
公司侧重点：抖音生活服务关注内容理解和转化效率；vivo 关注大模型测评；百度广告关注竞价和推荐；欣锐科技关注 Matlab 仿真和硬件协同；运营商关注数据分析与工程落地。

## 问答经验

### 问答经验 1
- 真实问题：如何将业务问题转化为算法问题？请举一个你做过的例子。
- 参考回答要点：先拆解业务目标，再判断是分类、回归、排序、推荐或检索问题；定义特征和标签，选择评估指标，说明数据限制、模型选择、实验结果和业务落地方式。
- 复盘反思：不要只说用了某个模型，要体现从业务到算法再到评估的完整路径。
- 公司/场景：百度广告、抖音生活服务、通用算法岗
- 考察能力：业务理解、算法建模、评估指标设计
- 难度：★★★★
- 来源说明：成员整理：算法岗公开面经与 JD 归纳，2024-2025
- 审核状态：可入库

## 能力模型
能力项：机器学习基础、特征与标签定义、评估指标、Python 数据处理、模型选择、业务问题转化、实验复盘、工程落地。
证据线索：竞赛记录、训练数据、特征工程、模型指标、实验对比、部署方式、业务收益或失败复盘。

## 面试追问
简历写算法模型名称不能直接等于掌握，应追问业务目标、问题类型、数据来源、特征、标签、指标、实验对比和上线约束。

## 重点关注
机器学习、深度学习、Python、特征工程、评估指标、业务转化、模型效果复盘。

## 面试官提示
优先验证候选人是否能把业务目标、数据、模型和评估指标连起来，而不是只背模型定义。
