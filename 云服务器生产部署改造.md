# AI-Interview 云服务器生产部署改造

## 改造原则

- 保留原有 `backend / frontend / admin` 三分结构
- 保留 `.env + Docker Compose + Alembic` 工作流
- 不改坏本地开发模式，`docker-compose.dev.yml` 和前端 `npm run dev` 仍可继续使用
- 新增生产文件，不再把 `PostgreSQL / Redis / FastAPI` 直接暴露到公网
- 与现有 `HireMate` 共存，优先复用其现有 `Caddy` 作为统一入口

## 推荐访问拓扑

- `https://<PRIMARY_DOMAIN>`：portal 项目选择页
- `https://<HIREMATE_DOMAIN>`：现有 HireMate
- `https://<AI_INTERVIEW_DOMAIN>`：AI-Interview 用户端
- `https://<AI_INTERVIEW_ADMIN_DOMAIN>`：AI-Interview 后台管理端

## 上线要点

1. 在服务器上创建共享网关网络：`docker network create shared_gateway`
2. 将现有 HireMate 的 `caddy` 服务接入该网络
3. 在 `ai-interview-backend` 中复制 `.env.production.example` 为 `.env`
4. 替换其中所有占位符
5. 使用 `docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build` 启动
6. 执行 `docker compose -f docker-compose.yml -f docker-compose.prod.yml exec app alembic upgrade head`
7. 如首次上线，执行 `docker compose -f docker-compose.yml -f docker-compose.prod.yml exec app python scripts/create_first_admin.py`
8. 将 Caddy 配置切换为统一入口配置，再 reload Caddy

## 回滚要点

- 优先保留数据库卷，不做 `down -v`
- 回滚应用层时，先恢复旧版 Caddyfile，再恢复上一版镜像或代码目录
- 数据库结构如需回滚，使用 Alembic 版本号回退，而不是手工删表
